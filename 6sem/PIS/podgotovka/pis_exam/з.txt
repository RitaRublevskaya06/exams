«Всё в одну папку» — что это значит на практике
В Go в одной папке = один package main = одна функция main().

Нельзя просто вставить:

целиком 1-19/main.go (там уже есть main и вызовы task5…task19), и
целиком 21/main.go (там второй main),
— будет ошибка: two main functions.

Правильно для билета «6 + 21»
Один файл main.go (или main.go + редко ещё файлы, но без второго main):

Из 1-19/main.go — скопировать:

нужные import (для 6: fmt);
функцию task6() целиком;
не копировать main() из 1-19.
Из 21/main.go — скопировать:

import для GORM (gorm.io/...);
структуру Product;
логику из main() переименовать, например, в task21() (всё тело main → функция task21()).
Свой один main():

func main() {
    task6()   // указатели
    task21()  // GORM
}
Импорты — один блок import со всем, что реально используется (объедини из двух файлов, дубликаты убери).

В папке лежат go.mod + go.sum (из подготовки), потому что 21 тянет GORM.

Порядок действий на экзамене (пошагово)
Создать папку, например D:\exam (как скажут).
Скопировать go.mod и go.sum из pis_exam в D:\exam.
Создать main.go.
Вставить код 6 и 21 по схеме выше (один main, две функции).
Убрать лишние комментарии, если просят.
В терминале:
cd D:\exam

Показать вывод: сначала блок задания 6, потом задания 21.



3. Зависимости — когда нужны
Задания	Нужен go mod?	Что импортируется
22, 23, 24
Нет
только стандартная библиотека
20, 21, 25–38
Да
sqlite, mux, gorm, websocket и т.д.
Если только 22–24 — достаточно:

cd путь\к\exam_task1
go run main.go
Если 20, 21, 25+ — один раз в папке с main.go:

go mod init exam
go mod tidy
go run main.go
go mod tidy сам подтянет пакеты из import (нужен интернет на экзамене, если ПК чистый).

Быстрый вариант: скопировать из подготовки файл go.mod (и лучше go.sum) в ту же папку, что и main.go, потом:

go mod download
go run main.go
