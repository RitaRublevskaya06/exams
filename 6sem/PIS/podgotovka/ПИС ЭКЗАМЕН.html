КРАТКИЙ ОТВЕТ ПО ВСЕМ ВОПРОСАМ (1-38) — ОПРЕДЕЛЕНИЯ, ПРИМЕРЫ, ОБЪЯСНЕНИЕ РАБОТЫ

ВОПРОС 1: GO: определение, свойства системы программирования

Определение: Go (Golang) — язык программирования, созданный в Google в 2009 году для системного и серверного программирования.

Как работает: Go компилируется в нативный машинный код, как C или C++. Вся стандартная библиотека и runtime вшиваются в один исполняемый файл. Это значит, что для запуска программы на сервере не нужно устанавливать никаких зависимостей — просто копируете бинарник и запускаете. При этом Go имеет встроенный планировщик горутин, который управляет параллельным выполнением.

Что значит: Статическая типизация означает, что тип каждой переменной известен на этапе компиляции, и компилятор проверяет, что вы не пытаетесь, например, умножить строку на число. Легкий деплоймент — это когда для развертывания приложения на сервере вам не нужен ни интерпретатор (как для Python), ни виртуальная машина (как для Java), ни куча DLL (как для .NET Framework).

Пример: Вы написали веб-сервер на Go, скомпилировали его в файл myapp.exe (или myapp на Linux). Этот файл весит около 10-15 МБ (включая весь runtime). Вы копируете его на сервер и запускаете — сервер работает. Для сравнения, приложение на Java требует установки JRE, на Python — интерпретатора и всех pip-пакетов.

ВОПРОС 2: GO: CLI: основные команды

Определение: CLI (Command Line Interface) — утилита go, через которую выполняются все операции с проектом.

Как работает: Когда вы вводите go run main.go, утилита сначала компилирует ваш код во временный исполняемый файл, а затем запускает его. Когда вы вводите go build, она создает постоянный исполняемый файл в текущей директории. go mod init создает файл go.mod, в котором прописывается имя модуля и версия Go.

Что значит: Команды делятся на категории: для работы с модулями (mod init, mod tidy, get), для компиляции (build, install, run), для тестирования (test), для форматирования (fmt), для анализа (vet).

Пример: Вы создаете новый проект. Сначала пишете go mod init myproject — появляется go.mod. Затем пишете go get github.com/gorilla/mux — пакет скачивается, и запись о нем добавляется в go.mod. Чтобы удалить неиспользуемые зависимости, выполняете go mod tidy. Для запуска приложения — go run main.go. Для сборки версии для продакшена — go build -o myapp.

ВОПРОС 3: GO: структура проекта

Определение: Структура проекта — это организация файлов и папок, соответствующая модульной системе Go.

Как работает: Go использует модульную систему. Модуль — это любая папка, содержащая файл go.mod. Внутри модуля вы можете создавать подпапки, каждая из которых является отдельным пакетом (package). Все .go файлы в одной папке должны объявлять один и тот же пакет. Имя пакета обычно совпадает с именем папки.

Что значит: Если у вас папка handlers, то все файлы в ней должны начинаться с package handlers. Пакет main особенный — из него собирается исполняемый файл, и он должен содержать функцию func main(). Файл go.mod хранит имя модуля (обычно URL вашего репозитория, например github.com/user/myproject), версию Go и список зависимостей с их версиями.

Пример: Структура веб-приложения: папка handlers/ — обработчики HTTP-запросов, папка repository/ — работа с базой данных, папка models/ — структуры данных, папка database/ — подключение к БД. Файл main.go в корне — точка входа. Папка internal/ для приватных пакетов, которые нельзя импортировать из других модулей.

ВОПРОС 4: GO: структура приложения

Определение: Структура приложения определяет, как организованы пакеты, где находится точка входа и как экспортируются символы.

Как работает: Каждый Go-файл начинается с объявления пакета (package main для точки входа). Затем идут импорты (import "fmt"), затем объявления констант, переменных, типов и функций. Имена с заглавной буквы экспортируются (видны из других пакетов), со строчной — приватны (видны только внутри своего пакета).

Что значит: Экспорт работает просто: если вы назвали функцию CalculateTotal, она будет доступна из других пакетов. Если назвали calculateTotal — нет. Функция init() выполняется автоматически до main() и может быть определена в любом пакете. Пакет может состоять из нескольких файлов — они все считаются частью одного пакета и могут обращаться к приватным символам друг друга.

Пример: Пакет math может быть разбит на add.go, subtract.go и multiply.go, но все они объявляют package math. В add.go есть функция Add (экспортируемая). В multiply.go есть функция multiply (приватная). Из другого пакета можно вызвать math.Add, но нельзя math.multiply. Функция init() в database.go может открывать соединение с БД при старте программы.

ВОПРОС 5: GO: фундаментальные типы, объявления, константы

Определение: Фундаментальные типы — встроенные типы данных: bool, string, int, float, byte, rune и другие.

Как работает: Каждая переменная в Go имеет тип, который определяет, какие значения она может принимать и какие операции с ней допустимы. Короткая форма объявления := выводит тип из значения справа. Константы вычисляются на этапе компиляции, поэтому они не могут быть результатом вызова функции (например, time.Now() нельзя присвоить константе).

Что значит: Нулевые значения — это значения по умолчанию: 0 для чисел, false для bool, "" для строк, nil для указателей и срезов. iota — это генератор последовательных целых чисел внутри блока const. При каждом новом const iota сбрасывается в 0. Строки в Go неизменяемы — любая операция вроде s = s + "!" создает новую строку, а не изменяет старую.

Пример: Если вы напишете var x int, то x будет равно 0. Если var s string — s будет "". const Pi = 3.14 — константа, вычисленная на этапе компиляции. const ( A = iota; B; C ) даст A=0, B=1, C=2. name := "John" — компилятор сам определит, что name имеет тип string.

ВОПРОС 6: GO: указатели, работа с указателями

Определение: Указатель — переменная, которая хранит адрес другой переменной в памяти. Обозначается символом *T (указатель на тип T).

Как работает: Когда вы берете адрес переменной с помощью &x, вы получаете указатель на x. Разыменование (доступ к значению) выполняется через *p. В отличие от C/C++, в Go нет арифметики указателей — вы не можете сделать p++, чтобы перейти к следующей ячейке памяти. Это сделано для безопасности.

Что значит: Указатели нужны, когда вы хотите, чтобы функция изменила переменную, переданную ей как аргумент. Без указателей Go передает копию значения. Нулевое значение указателя — nil. Разыменование nil-указателя вызывает панику (аналог segmentation fault). В Go есть new(T), который выделяет память для T и возвращает указатель на нулевое значение.

Пример: У вас есть функция func increment(p *int) { *p = *p + 1 }. Вызов increment(&x) увеличит x на 1. Если бы вы передали increment(x), функция получила бы копию и x не изменился бы. Указатели на структуры автоматически разыменовываются: если p — указатель на структуру, можно писать p.Name вместо (*p).Name.

ВОПРОС 7: GO: массивы, многомерные массивы

Определение: Массив — фиксированная последовательность элементов одного типа. Длина массива является частью его типа.

Как работает: При объявлении [5]int компилятор выделяет 5 ячеек памяти подряд. Размер массива известен на этапе компиляции. Когда вы передаете массив в функцию, передается копия всего массива (все элементы копируются). При присваивании одного массива другому также копируются все элементы.

Что значит: [3]int и [5]int — это разные типы, их нельзя присваивать друг другу. Доступ к элементам по индексу с проверкой границ — если вы попытаетесь обратиться к arr[5] для массива из 3 элементов, программа упадет (паника). Многомерные массивы — это массивы массивов: [3][4]int — 3 строки по 4 элемента каждая, всего 12 элементов в непрерывной памяти.

Пример: [5]int{1,2,3,4,5} — массив из пяти чисел. var matrix [3][3]int — квадратная матрица 3x3. Доступ к элементу: matrix[1][2]. Чтобы обойти все элементы, нужны два вложенных цикла. Для передачи большого массива в функцию эффективнее использовать указатель: func process(arr *[1000000]int) — тогда копироваться будет только указатель (8 байт), а не 8 мегабайт данных.

ВОПРОС 8: GO: срезы (slice), работа со срезами

Определение: Срез (slice) — динамическое представление массива. Содержит указатель на массив, длину (len) и емкость (cap).

Как работает: Когда вы создаете срез, Go создает небольшой объект (24 байта на 64-битной системе) — указатель на массив, длину и емкость. При append срез смотрит: если длины хватает (len < cap), он просто увеличивает len и записывает элемент в существующий массив. Если емкость исчерпана, он создает новый массив большего размера (обычно в 2 раза), копирует туда старые элементы и добавляет новый.

Что значит: Срезы — ссылочный тип. Если вы измените элемент среза, вы измените общий базовый массив. Другие срезы, ссылающиеся на тот же массив, увидят это изменение. Создать срез можно из массива (arr[1:4]), с помощью литерала ([]int{1,2,3}) или через make([]int, 5, 10). make сразу создает массив нужной емкости, что эффективнее многократного append.

Пример: s := []int{1,2,3} — срез, указывающий на массив из трех элементов. s = append(s, 4) — теперь len=4, cap может быть 6 (если потребовалось перевыделение). s2 := s[1:3] — второй срез, ссылающийся на элементы 2 и 3 того же массива. Если вы измените s2[0] = 100, то s[1] тоже станет 100.

ВОПРОС 9: GO: ассоциативные массивы (map), работа с map

Определение: Map (словарь) — неупорядоченная коллекция пар ключ-значение. Ключ должен быть сравнимым типом.

Как работает: Внутри map реализован как хэш-таблица. Когда вы добавляете элемент, Go вычисляет хэш от ключа, определяет номер "корзины" (bucket), и сохраняет пару ключ-значение в этой корзине. При поиске по ключу снова вычисляется хэш, находится нужная корзина, и в ней линейно ищется ключ. Порядок итерации по map не гарантирован — он может меняться от запуска к запуску намеренно.

Что значит: Нулевое значение map — nil. В nil-карту нельзя добавлять элементы — будет паника. Создавать map нужно через make или литерал. Проверка существования ключа выполняется с помощью value, ok := m[key]. ok будет true, если ключ есть. Если ключа нет, value получает нулевое значение. delete(m, key) удаляет элемент (если ключа нет, ничего не происходит).

Пример: ages := map[string]int{"Alice": 30, "Bob": 25} — map с ключами-строками и значениями-числами. ages["Charlie"] = 35 — добавить. age, exists := ages["Alice"] — age=30, exists=true. age, exists := ages["Nobody"] — age=0, exists=false. delete(ages, "Bob") — удалить. Обход: for name, age := range ages { fmt.Println(name, age) } — порядок случайный.

ВОПРОС 10: GO: пользовательские типы, работа с пользовательскими типами

Определение: Пользовательский тип — новый тип, созданный на основе существующего с помощью ключевого слова type.

Как работает: Когда вы пишете type Celsius float64, вы создаете новый тип с именем Celsius. Базовый тип — float64, но Celsius и float64 — разные типы. Нельзя присвоить float64 переменной типа Celsius без явного преобразования. Для пользовательского типа можно определять методы — функции с receiver-аргументом.

Что значит: Это позволяет добавить поведение (методы) к любому типу, даже к встроенному. Например, можно добавить метод .ToFahrenheit() к типу Celsius. При создании нового типа он наследует свойства базового типа (можно использовать операции +, -, *, /, если они определены для базового типа), но методы не наследуются — их нужно определять заново.

Пример: type Counter int. Теперь Counter — отдельный тип. Можно написать func (c Counter) Increment() Counter { return c + 1 }. Нельзя написать var x int = Counter(5) — нужно явное преобразование: x := int(Counter(5)). Пользовательские типы улучшают читаемость и типобезопасность — функция, принимающая Celsius, не примет просто float64 по ошибке.

ВОПРОС 11: GO: алиасы, работа с алиасами

Определение: Алиас (псевдоним) — альтернативное имя для существующего типа. Синтаксис: type Alias = ExistingType.

Как работает: В отличие от пользовательского типа, алиас не создает новый тип. type MyString = string означает, что MyString и string — это один и тот же тип. Вы можете присваивать MyString переменной типа string и наоборот без приведения. Алиас наследует все методы исходного типа.

Что значит: Алиасы полезны при рефакторинге — вы можете переименовать тип, но оставить алиас для обратной совместимости. Также алиасы используются для сокращения длинных имен типов (например, type HandlerFunc = func(http.ResponseWriter, *http.Request)). В отличие от пользовательского типа (type MyType T), алиас (type MyType = T) — это просто другое имя для того же типа.

Пример: type UserID = string. Теперь var uid UserID = "abc" и var s string = uid работают без приведения. Если вы измените реальный тип на int, код с алиасом может сломаться — алиас не дает защиты. type ErrorHandler = func(error) — алиас для типа функции, удобно для передачи в качестве аргумента.

ВОПРОС 12: GO: структуры, работа со структурами

Определение: Структура (struct) — составной тип, объединяющий поля разных типов. Аналог класса без методов-наследования.

Как работает: Структура занимает непрерывный участок памяти, где поля расположены одно за другим (с возможным выравниванием). При передаче структуры в функцию копируются все её поля. Чтобы избежать копирования, передают указатель на структуру. Методы можно определять на структуре, используя receiver — это может быть значение ((p Person)) или указатель ((p *Person)).

Что значит: Встраивание (embedding) — это когда вы включаете одну структуру в другую без указания имени поля. При этом поля встроенной структуры становятся доступны как будто они являются полями внешней структуры. Это композиция, а не наследование. Теги (tags) — это строки после поля, которые используются пакетами вроде encoding/json для маппинга.

Пример: type Person struct { Name string; Age int }. Встраивание: type Employee struct { Person; Salary int } — теперь Employee имеет поля Name, Age, Salary. Теги: type User struct { ID int json:"id"; Password string json:"-" } — при сериализации в JSON поле ID станет "id", поле Password будет пропущено.

ВОПРОС 13: GO: строки, работа со строками

Определение: Строка — неизменяемая последовательность байт, представляющая текст в кодировке UTF-8.

Как работает: Строка в Go — это структура из двух полей: указатель на область памяти и длина в байтах. Строки неизменяемы: когда вы "изменяете" строку (например, s = s + "!"), создается новая строка, а старая остается в памяти до сборки мусора. Длина строки len(s) возвращает количество байт, а не символов. Для работы с символами (рунами) используется []rune(s) или обход через range.

Что значит: Поскольку строки в UTF-8, символы могут занимать от 1 до 4 байт. Русская буква "Я" занимает 2 байта. Срез строки s[i:j] работает на уровне байт — если вы обрежете посередине многобайтового символа, получите некорректную UTF-8 последовательность. Пакет strings предоставляет функции для работы со строками без учета рун. strings.Builder эффективен для многократной конкатенации — он выделяет буфер и наращивает его по мере необходимости, избегая множественных аллокаций.

Пример: s := "Привет", len(s) вернет 12 (6 символов × 2 байта). s[0] вернет первый байт (часть буквы "П"), что некорректно. Правильно: runes := []rune(s) — массив из 6 рун. strings.ToUpper(s) — переведет в верхний регистр (для латиницы). strings.Split("a,b,c", ",") — разделит на ["a","b","c"].

ВОПРОС 14: GO: операции

Определение: Операции — символы, выполняющие действия над значениями: арифметические, битовые, логические, сравнения, присваивания.

Как работает: У каждого оператора есть приоритет и ассоциативность. ++ и -- работают только постфиксно (нельзя ++x), и они не возвращают значение (нельзя y = x++). Операторы сравнения работают для любых comparable типов. Логические операторы && и || используют short-circuit evaluation — если левый операнд определяет результат, правый не вычисляется.

Что значит: && (И) — если левое выражение ложно, правое не вычисляется. || (ИЛИ) — если левое истинно, правое не вычисляется. Это важно, если правое выражение может вызвать панику (например, разыменование указателя). Сдвиги << и >> для знаковых типов — арифметические (знак сохраняется), для беззнаковых — логические (заполняются нулями). := — короткое объявление, создает новую переменную и присваивает ей значение.

Пример: x := 5; y := x++ — так нельзя, x++ не возвращает значение. if p != nil && *p > 0 — безопасно: если p == nil, второе условие не проверяется. a := 10; b := 3; a % b — остаток от деления, результат 1. flags := 0b1010; flags |= 0b0100 — установка бита. name := "John" — короткое объявление.

ВОПРОС 15: GO: конструкции if, switch

Определение: if и switch — условные операторы для ветвления выполнения программы.

Как работает: В if можно включить короткое объявление переменной, которая будет видна только внутри блока if и связанных else. Switch не требует break (выход из case происходит автоматически). Для перехода к следующему case нужно явно указать fallthrough. Switch может быть без выражения — тогда он работает как if-else цепочка. Type switch проверяет тип интерфейса.

Что значит: Переменная, объявленная в if, не "засоряет" внешнюю область видимости. Это хорошая практика для обработки ошибок. В switch case могут содержать множественные значения через запятую. fallthrough выполняет следующий case без проверки его условия — это редко нужно и может привести к багам. Type switch — мощный механизм для работы с интерфейсами.

Пример: if err := doSomething(); err != nil { return err } — переменная err живет только внутри if. switch day { case "Mon","Tue","Wed": fmt.Println("Weekday") } — множественные case. switch { case score >= 90: grade = "A"; case score >= 80: grade = "B" } — switch без выражения. switch v.(type) { case int: fmt.Println("int") } — type switch.

ВОПРОС 16: GO: циклы

Определение: В Go только один оператор цикла — for. Он может использоваться в трех формах: классический, как while, и бесконечный.

Как работает: Классический for состоит из трех частей: инициализация, условие, последействие. Любую часть можно опустить. Цикл for condition {} — это while-цикл. for {} — бесконечный цикл. range позволяет итерироваться по массивам, срезам, строкам (по рунам), map (случайный порядок) и каналам.

Что значит: В Go нет отдельных операторов while и do-while. break прерывает цикл, continue переходит к следующей итерации. Можно использовать метки для выхода из внешних вложенных циклов. При итерации по строке через range, первый индекс — позиция в байтах, второй — руна (символ). Это удобно, но осторожно: индекс не увеличивается на 1 для многобайтовых символов.

Пример: for i := 0; i < 10; i++ { } — классический. for x < 10 { x++ } — как while. for { if done { break } } — бесконечный. for i, ch := range "Hello" { fmt.Printf("%d: %c\n", i, ch) } — range по строке, индексы 0,1,2,3,4. for key, val := range m { } — range по map.

ВОПРОС 17: GO: функции, применение функций

Определение: Функция — именованный блок кода с параметрами и возвращаемыми значениями. В Go функции — first-class citizens.

Как работает: Функции могут возвращать несколько значений. Это часто используется для возврата результата и ошибки. Функции можно присваивать переменным, передавать как аргументы и возвращать из других функций (замыкания). defer откладывает выполнение функции до выхода из текущей функции — полезно для закрытия файлов, разблокировки мьютексов.

Что значит: Множественные возвращаемые значения — одна из ключевых особенностей Go. Вместо того чтобы возвращать структуру с полем error или использовать исключения, Go использует явный возврат (result, error). Замыкания позволяют функции "запоминать" переменные из внешней области видимости. defer выполняется даже при панике — гарантирует освобождение ресурсов. Вариативные параметры (...T) позволяют передавать любое количество аргументов.

Пример: func divide(a, b int) (int, error) { if b == 0 { return 0, errors.New("zero") }; return a/b, nil } — возврат двух значений. defer file.Close() — закроет файл при выходе из функции. counter := func() func() int { count := 0; return func() int { count++; return count } }() — замыкание, создает счетчик. func sum(nums ...int) int { } — вариативная функция, sum(1,2,3), sum([]int{1,2}...).

ВОПРОС 18: GO: горутины, применение горутин

Определение: Горутина (goroutine) — легковесный поток выполнения, управляемый runtime Go. Запускается ключевым словом go.

Как работает: Когда вы пишете go function(), Go создает новую горутину — структуру с маленьким стеком (около 2 КБ). Планировщик Go распределяет горутины на системные потоки (по умолчанию количество потоков равно количеству CPU). Если горутина блокируется (например, на вводе-выводе), планировщик переключает поток на другую горутину. Это позволяет десяткам тысяч горутин выполняться на нескольких потоках.

Что значит: Горутины намного легче потоков ОС. Создать 10000 горутин — обычное дело, 10000 потоков ОС — проблема. Каналы (chan) — основной способ общения между горутинами. Отправка в канал блокирует горутину, пока кто-то не прочитает. Чтение из пустого канала блокирует. select позволяет ожидать события на нескольких каналах.

Пример: go processData(data) — запуск функции в горутине. ch := make(chan int); go func() { ch <- 42 }(); x := <-ch — коммуникация через канал. select { case msg := <-ch1: fmt.Println(msg); case <-time.After(1*time.Second): fmt.Println("timeout") } — ожидание с таймаутом. sync.WaitGroup для ожидания группы горутин: wg.Add(1); go func() { defer wg.Done(); work() }(); wg.Wait().

ВОПРОС 19: GO: работа с файловой системой

Определение: Работа с файловой системой включает чтение, запись, создание, удаление файлов и директорий через пакеты os, io, bufio.

Как работает: os.Open открывает файл только для чтения, os.Create создает файл для записи (перезаписывает существующий), os.OpenFile дает больше контроля. При чтении через Read, вы передаете буфер (срез байт), и функция возвращает количество прочитанных байт. При достижении конца файла возвращается io.EOF. io.Copy читает из источника и пишет в приемник буферами по умолчанию (32 КБ), что эффективно.

Что значит: Всегда нужно закрывать файлы через defer file.Close() — иначе утечка файловых дескрипторов. os.ReadFile и os.WriteFile удобны для небольших файлов (читают/пишут целиком в память). Для больших файлов используйте буферизированное чтение через bufio.Reader. filepath пакет обеспечивает кросс-платформенную работу с путями (filepath.Join для объединения, filepath.Walk для обхода).

Пример: data, err := os.ReadFile("config.json") — прочитать весь файл. os.WriteFile("out.txt", []byte("data"), 0644) — записать. src, _ := os.Open("src.txt"); dst, _ := os.Create("dst.txt"); io.Copy(dst, src) — скопировать с буферизацией. scanner := bufio.NewScanner(file); for scanner.Scan() { fmt.Println(scanner.Text()) } — построчное чтение. os.MkdirAll("path/to/dir", 0755) — создать директорию с родителями.

ВОПРОС 20: database/sql для работы с БД

Определение: database/sql — стандартный пакет, предоставляющий универсальный интерфейс для работы с реляционными базами данных через драйверы.

Как работает: Вы открываете "базу данных" с помощью sql.Open(driverName, dataSourceName), получаете *sql.DB. Это не соединение, а пул соединений. При первом запросе пул создает соединение, выполняет запрос, и возвращает соединение в пул. Параметры SetMaxOpenConns, SetMaxIdleConns контролируют размер пула. Транзакция создается через db.Begin(), возвращает *sql.Tx.

Что значит: sql.DB безопасен для конкурентного использования — можно иметь один глобальный объект для всего приложения. Драйвер регистрируется через _ "github.com/microsoft/go-mssqldb" (импорт только для init()). db.Ping() проверяет, жива ли БД. Prepare создает подготовленный запрос — полезен для многократного выполнения с разными параметрами. Query vs Exec: Query возвращает строки (для SELECT), Exec — результат (для INSERT/UPDATE/DELETE).

Пример: db, _ := sql.Open("sqlserver", "server=localhost;database=test"). rows, _ := db.Query("SELECT id, name FROM users WHERE age > ?", 18). defer rows.Close() обязательно! for rows.Next() { rows.Scan(&id, &name) }. tx, _ := db.Begin(); tx.Exec("UPDATE ..."); tx.Commit(). result, _ := db.Exec("INSERT INTO users(name) VALUES(?)", "John"); id, _ := result.LastInsertId().

ВОПРОС 21: применение фреймворка GORM

Определение: GORM — ORM-библиотека, преобразующая GO-структуры в SQL-запросы и наоборот, автоматизирующая CRUD-операции.

Как работает: Вы определяете структуру с тегами (например, gorm:"primaryKey"). GORM при миграции анализирует структуру и генерирует SQL для создания/изменения таблицы. При вызове db.Create(&user) GORM конструирует INSERT-запрос, выполняя хуки перед вставкой. При вызове db.First(&user, 1) генерируется SELECT. При обновлении — UPDATE. GORM использует reflect для чтения полей структур.

Что значит: Хуки — методы, вызываемые до/после операций: BeforeCreate, AfterCreate, BeforeUpdate, AfterUpdate, BeforeDelete, AfterDelete. Они могут изменять данные или возвращать ошибку для прерывания. Preload решает проблему N+1 запросов: вместо отдельного запроса для каждой ассоциации, GORM делает один запрос с JOIN или несколько запросов (но контролируемо). Soft delete — если в модели есть DeletedAt, Delete не удаляет запись, а устанавливает deleted_at.

Пример: type User struct { gorm.Model; Name string; Email string gorm:"uniqueIndex" }. db.AutoMigrate(&User{}) — создаст таблицу. db.Create(&User{Name: "John"}) — вставка. db.First(&user, 1) — поиск по PK. db.Where("name = ?", "John").Find(&users) — поиск по условию. db.Model(&user).Update("name", "Jane") — обновление. db.Delete(&user) — мягкое удаление.

ВОПРОС 22: простейший http-сервер (net/http)

Определение: HTTP-сервер на базе net/http — встроенный пакет, позволяющий запустить веб-сервер с минимальным кодом.

Как работает: http.HandleFunc("/path", handler) регистрирует обработчик в DefaultServeMux. http.ListenAndServe(":8080", nil) создает TCP-слушатель на порту 8080. При входящем запросе сервер создает новую горутину и вызывает обработчик. Горутина обрабатывает запрос и завершается. Ответ отправляется через http.ResponseWriter, запрос доступен через *http.Request.

Что значит: Каждый запрос обрабатывается в отдельной горутине — параллелизм из коробки. fmt.Fprintf(w, "Hello") отправляет ответ. r.URL.Path содержит путь запроса. r.Method содержит HTTP-метод. http.Error(w, "message", code) — удобная отправка ошибки. Сервер блокирует выполнение программы — код после ListenAndServe не выполняется (если не в горутине).

Пример: http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) { fmt.Fprintf(w, "Hello, %s", r.URL.Path) }). http.ListenAndServe(":8080", nil). При запросе GET /users ответит "Hello, /users". Каждый запрос в своей горутине — если обработчик спит 10 секунд, другие запросы все равно обрабатываются параллельно.

ВОПРОС 23: маршрутизация, вывод log

Определение: Маршрутизация — сопоставление URL с обработчиками. log — пакет для потокобезопасного логирования с временными метками.

Как работает: http.NewServeMux() создает мультиплексор. mux.HandleFunc("/users", usersHandler) регистрирует обработчик. При запросе мультиплексор ищет самый длинный совпадающий префикс. log.Printf("message %s", arg) выводит сообщение в stderr с временной меткой. log.Fatal выводит и завершает программу.

Что значит: DefaultServeMux — глобальный, лучше создавать свой. log потокобезопасен — можно логировать из горутин без дополнительной синхронизации. log.SetFlags настраивает вывод. log.SetOutput(file) направляет логи в файл. log.Println автоматически добавляет перевод строки. Используйте log вместо fmt для серверов — он дает контекст (время, файл, строку).

Пример: mux := http.NewServeMux(); mux.HandleFunc("/api/users", getUsers); mux.HandleFunc("/api/posts", getPosts). log.Printf("Request: %s %s from %s", r.Method, r.URL.Path, r.RemoteAddr). log.SetFlags(log.LstdFlags | log.Lshortfile) — добавит файл и строку. log.SetOutput(os.Stdout) — логи в консоль.

ВОПРОС 24: POST-запрос, обработка JSON

Определение: Обработка POST-запросов с JSON-телом — чтение JSON из r.Body, декодирование в структуру, отправка JSON-ответа.

Как работает: json.NewDecoder(r.Body).Decode(&data) читает тело запроса как JSON и преобразует в GO-структуру. При этом автоматически учитываются теги json:"fieldName". json.NewEncoder(w).Encode(data) кодирует структуру в JSON и отправляет. w.Header().Set("Content-Type", "application/json") устанавливает правильный заголовок.

Что значит: Теги структур позволяют переименовывать поля при сериализации: Name string json:"name"``. omitempty — не включать поле, если оно нулевое. json:"-" — полностью игнорировать поле. Всегда проверяйте ошибку декодирования — клиент может прислать невалидный JSON. r.Body закрывается автоматически сервером. После отправки ответа закрывать ничего не нужно.

Пример: Структура type User struct { Name string json:"name"; Age int json:"age,omitempty" }. В обработчике: var user User; err := json.NewDecoder(r.Body).Decode(&user); if err != nil { http.Error(w, err.Error(), 400); return }. Ответ: w.Header().Set("Content-Type", "application/json"); json.NewEncoder(w).Encode(user).

ВОПРОС 25: gorilla/mux: маршрутизация, JSON

Определение: gorilla/mux — мощный роутер с поддержкой переменных пути, методов HTTP, регулярных выражений и middleware.

Как работает: r := mux.NewRouter() создает роутер. r.HandleFunc("/users/{id}", handler).Methods("GET") регистрирует маршрут с переменной id. При запросе GET /users/123 роутер извлекает id=123 и передает в обработчик через mux.Vars(r). JSON обрабатывается так же, как в net/http.

Что значит: В отличие от net/http, mux умеет различать методы без ручной проверки в обработчике. Переменные пути доступны через vars := mux.Vars(r) как map[string]string. Можно использовать регулярные выражения: {id:[0-9]+}. Subrouters позволяют группировать маршруты и применять middleware к группе. Маршрутизатор реализует интерфейс http.Handler, так что его можно передать в http.ListenAndServe.

Пример: r := mux.NewRouter(); r.HandleFunc("/api/users/{id}", getUser).Methods("GET"); r.HandleFunc("/api/users", createUser).Methods("POST"). В обработчике: vars := mux.Vars(r); id := vars["id"]. JSON: json.NewDecoder(r.Body).Decode(&user). http.ListenAndServe(":8080", r).

ВОПРОС 26: обработка x-www-form-urlencoded

Определение: application/x-www-form-urlencoded — формат HTML-форм, где данные передаются как key1=value1&key2=value2 в теле POST-запроса.

Как работает: r.ParseForm() читает тело запроса (и query-параметры) и заполняет r.Form (map). r.FormValue("key") возвращает значение для ключа (первое, если несколько). Пробелы кодируются как +, спецсимволы как %XX. ParseForm нужно вызвать явно, если вы хотите проверить ошибку парсинга.

Что значит: FormValue автоматически вызывает ParseForm, но скрывает ошибки. Для надежности вызывайте ParseForm явно. Для множественных значений (checkbox) используйте r.Form["key"] — вернет []string. Не используйте этот формат для загрузки файлов (для этого есть multipart/form-data). После ParseForm данные доступны и в r.PostForm (только тело) и в r.Form (тело + query).

Пример: HTML-форма: <form method="POST" action="/submit"><input name="name"><input type="checkbox" name="hobby" value="reading">. Обработчик: r.ParseForm(); name := r.FormValue("name"); hobbies := r.Form["hobby"]. Ответ: fmt.Fprintf(w, "Name: %s, Hobbies: %v", name, hobbies).

ВОПРОС 27: POST, JSON через gorilla/mux

Определение: Комбинация gorilla/mux для маршрутизации и стандартной обработки JSON для POST-запросов.

Как работает: mux маршрутизирует запрос по методу POST и пути, вызывает обработчик. В обработчике читается JSON из тела запроса, валидируется, выполняется бизнес-логика, отправляется JSON-ответ. mux не влияет на обработку JSON — это стандартный пакет encoding/json.

Что значит: mux берет на себя проверку метода (Methods("POST")) — в обработчике можно не проверять. Это делает код чище. Переменные пути можно комбинировать с JSON: например, PUT /users/{id} с JSON-телом для обновления. Всегда возвращайте правильные HTTP статусы: 201 Created для создания, 200 OK для успеха, 400 Bad Request для ошибок валидации.

Пример: r.HandleFunc("/api/users", createUser).Methods("POST"). В createUser: var user User; json.NewDecoder(r.Body).Decode(&user); if user.Name == "" { http.Error(w, "name required", 400); return }; save(user); w.WriteHeader(201); json.NewEncoder(w).Encode(user).

ВОПРОС 28: обработка path-параметров в mux

Определение: Path-параметры — переменные части URL, определяемые в маршруте как {имя} и извлекаемые через mux.Vars(r).

Как работает: В маршруте "/users/{id}/posts/{postId}" фигурные скобки обозначают переменные. При совпадении запроса GET /users/123/posts/456 mux извлекает id=123, postId=456. Обработчик вызывает vars := mux.Vars(r) и получает map[string]string{"id":"123","postId":"456"}.

Что значит: Path-параметры — часть URL, они обязательны (если переменная есть в маршруте, без нее не совпадет). Можно добавлять регулярные выражения: {id:[0-9]+} — только цифры. {name:[a-z]+} — только строчные латинские буквы. Порядок регистрации важен: сначала более конкретные маршруты, потом общие. "/users/profile" нужно зарегистрировать до "/users/{id}", иначе profile станет значением id.

Пример: r.HandleFunc("/products/{category}/{id}", productHandler). В обработчике: vars := mux.Vars(r); category := vars["category"]; id := vars["id"]. r.HandleFunc("/users/{id:[0-9]+}", getUserByID) — только цифровые ID. r.HandleFunc("/files/{name:.+\\.txt}", getTextFile) — только .txt файлы.

ВОПРОС 29: обработка query-параметров

Определение: Query-параметры — параметры после ? в URL (например, /search?q=golang&page=2), извлекаются через r.URL.Query().

Как работает: r.URL.Query() возвращает url.Values — тип map[string][]string. q := query.Get("q") возвращает первое значение для ключа "q" или пустую строку. all := query["id"] возвращает все значения (например, ?id=1&id=2&id=3). Все параметры — строки, числа нужно преобразовывать через strconv.Atoi.

Что значит: Query-параметры всегда опциональны — их отсутствие не ошибка. mux не поддерживает их в определении маршрутов (в отличие от path-параметров). Используйте query-параметры для фильтрации, пагинации, сортировки. Для пагинации задавайте разумные значения по умолчанию (page=1, limit=10). Ограничивайте максимальный limit, чтобы клиент не запрашивал миллион записей.

Пример: GET /api/users?page=2&limit=10&sort=name&active=true. Обработчик: q := r.URL.Query(); page, _ := strconv.Atoi(q.Get("page")); if page < 1 { page = 1 }; limit, _ := strconv.Atoi(q.Get("limit")); if limit > 100 { limit = 100 }; sort := q.Get("sort"); active := q.Get("active") == "true".

ВОПРОС 30: скачивание статических файлов

Определение: Отдача статических файлов (изображений, CSS, JS, PDF) через HTTP с помощью http.ServeFile или http.FileServer.

Как работает: http.ServeFile(w, r, filePath) открывает файл, определяет Content-Type по расширению, отправляет заголовки и содержимое. http.FileServer(http.Dir("./static")) создает обработчик, который отдает файлы из директории. http.StripPrefix удаляет префикс из URL перед передачей в FileServer.

Что значит: http.ServeFile безопасна от directory traversal (попыток выйти из директории через ../), но только если переданный путь не содержит пользовательского ввода. Если путь содержит .., нужно очистить через filepath.Clean и проверить префикс. FileServer автоматически отдает index.html при запросе директории. Установка заголовка Content-Disposition: attachment заставляет браузер скачать файл, а не отображать.

Пример: r.PathPrefix("/static/").Handler(http.StripPrefix("/static/", http.FileServer(http.Dir("./static/")))). Для скачивания: r.HandleFunc("/download/{file}", func(w http.ResponseWriter, r *http.Request) { file := mux.Vars(r)["file"]; w.Header().Set("Content-Disposition", "attachment; filename="+file); http.ServeFile(w, r, "./uploads/"+file) }).

ВОПРОС 31: обработка multipart/form-data

Определение: multipart/form-data — формат для загрузки файлов, где данные разделяются boundary-строкой, а файлы передаются в виде бинарных блоков.

Как работает: r.ParseMultipartForm(maxMemory) читает тело запроса. Данные форм (текстовые поля) загружаются в память. Файлы, если их размер не превышает maxMemory, также загружаются в память, иначе сохраняются во временные файлы на диске. r.FormFile("fieldname") возвращает multipart.File (интерфейс io.Reader) и *multipart.FileHeader с метаданными файла.

Что значит: ParseMultipartForm нужно вызывать перед доступом к файлам. Лимит maxMemory — компромисс: маленькое значение → больше временных файлов (медленнее), большое → больше RAM (риск исчерпания). После вызова FormFile файл открыт, его нужно закрыть через defer file.Close(). Для множественных файлов используйте r.MultipartForm.File["fieldname"]. BOM (Byte Order Mark) может присутствовать в текстовых файлах, особенно из Windows.

Пример: r.ParseMultipartForm(10 << 20) — 10 MB в памяти. file, header, err := r.FormFile("upload"). defer file.Close(). dst, _ := os.Create("./uploads/" + header.Filename); io.Copy(dst, file). Ответ: json.NewEncoder(w).Encode(map[string]interface{}{"filename": header.Filename, "size": header.Size}).

ВОПРОС 32: маршрутизация по query-параметрам

Определение: Маршрутизация по query-параметрам — изменение поведения обработчика в зависимости от значений GET-параметров.

Как работает: mux не умеет маршрутизировать на основе query-параметров. Вместо этого создается один маршрут, а внутри обработчика анализируются параметры через r.URL.Query(). На основе значений параметров вызываются разные ветки кода. Для сквозной логики (например, выбор формата ответа) можно использовать middleware, который помещает параметр в контекст запроса.

Что значит: Query-параметры предназначены для фильтрации, сортировки и пагинации, а не для определения ресурса. Поэтому их отсутствие в маршрутизаторе — это не недостаток, а осознанное архитектурное решение. Если вам нужно разное поведение, основанное на параметре, используйте switch по значению параметра внутри обработчика.

Пример: Маршрут: r.HandleFunc("/api/data", dataHandler). Обработчик: format := r.URL.Query().Get("format"); switch format { case "json": serveJSON(w, data); case "xml": serveXML(w, data); default: serveJSON(w, data) }. Middleware: ctx := context.WithValue(r.Context(), "format", format); r = r.WithContext(ctx), затем в обработчике format := r.Context().Value("format").

ВОПРОС 33: JSON-RPC сервер

Определение: JSON-RPC — протокол удаленного вызова процедур, использующий JSON для кодирования вызовов и ответов.

Как работает: Клиент отправляет POST-запрос с JSON-объектом, содержащим jsonrpc: "2.0", method (имя процедуры), params (параметры) и id (идентификатор). Сервер декодирует запрос, вызывает соответствующую функцию, кодирует результат в JSON и отправляет ответ. Если id отсутствует, это уведомление — ответ не отправляется.

Что значит: Параметры могут быть позиционными (массив) или именованными (объект). Коды ошибок стандартизованы: -32700 (Parse error), -32601 (Method not found) и т.д. Можно отправлять пакет запросов — массив запросов, ответ — массив ответов в том же порядке. Транспорт — обычно HTTP или WebSocket. JSON-RPC прост и легковесен, но менее автоматизирован, чем gRPC.

Пример: Запрос: {"jsonrpc":"2.0","method":"add","params":[5,3],"id":1}. Ответ: {"jsonrpc":"2.0","result":8,"id":1}. Ошибка: {"jsonrpc":"2.0","error":{"code":-32601,"message":"Method not found"},"id":1}. Уведомление: {"jsonrpc":"2.0","method":"ping","params":[]} — без id, ответа не будет.

ВОПРОС 34-35: WebSocket-сервер

Определение: WebSocket — протокол полнодуплексной связи поверх TCP, позволяющий серверу и клиенту обмениваться сообщениями в реальном времени.

Как работает: Начинается с HTTP-запроса с заголовком Upgrade: websocket. Сервер отвечает 101 Switching Protocols. После этого соединение "апгрейдится", и обе стороны могут отправлять сообщения (кадры) в любое время. Кадры имеют тип: текст (UTF-8), бинарные данные, ping/pong (heartbeat), close. Для поддержания соединения используются ping/pong.

Что значит: WebSocket поддерживает низкую задержку (30-100 мс RTT) и подходит для чатов, игр, биржевых данных. В отличие от HTTP, нет overhead на заголовки для каждого сообщения. gorilla/websocket — популярная библиотека. upgrader.Upgrade переводит HTTP-соединение в WebSocket. conn.ReadMessage и conn.WriteMessage для обмена. Код закрытия 1000 — нормальное закрытие, 1006 — аномальное.

GO-клиент пример: conn, _, _ := websocket.DefaultDialer.Dial("ws://localhost:8080/ws", nil). conn.WriteMessage(websocket.TextMessage, []byte("hello")). _, msg, _ := conn.ReadMessage(). JS-клиент: let ws = new WebSocket("ws://localhost:8080/ws"); ws.onopen = () => ws.send("hello"); ws.onmessage = (e) => console.log(e.data);

ВОПРОС 36: WebDAV-сервер

ВОПРОС 37: GraphQL-сервер

Определение: GraphQL — язык запросов к API, где клиент сам описывает, какие поля ему нужны, и получает ровно их.

Как работает: Клиент отправляет POST-запрос с JSON, содержащим query (строка запроса на языке GraphQL) и опционально variables. Сервер парсит запрос, проверяет его по схеме, вызывает ресолверы для каждого запрошенного поля и собирает ответ в той же структуре, что и запрос. Ресолверы могут обращаться к базе данных, другим API и т.д.

Что значит: Схема определяет типы, запросы (Query — чтение) и мутации (Mutation — изменение). Ресолвер — функция, которая возвращает данные для поля. Если поле — объект, ресолвер может вернуть промис или другой объект, и вложенные поля будут ресолвиться дальше. GraphQL решает проблему over-fetching и under-fetching REST API. Один endpoint для всех операций. Есть встроенная документация через introspection.

Пример схемы: type User { id: ID!; name: String!; email: String! } type Query { user(id: ID!): User; users: [User!]! }. Запрос: query { users { id name } }. Ответ: {"data": {"users": [{"id": "1", "name": "John"}]}}. Мутация: mutation { createUser(name: "Jane") { id name } }.
