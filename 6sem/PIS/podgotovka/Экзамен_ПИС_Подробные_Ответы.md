## Определение языка Go

Go (часто называемый Golang) — это компилируемый, статически типизированный язык программирования общего назначения, разработанный в компании Google в 2007–2009 годах командой во главе с Робертом Гризмером, Робом Пайком и Кеном Томпсоном (последний два — создатели языка C и операционной системы UNIX). Публичный релиз состоялся в 2009 году, первая стабильная версия Go 1.0 вышла в марте 2012 года.

Язык был создан в ответ на недостатки существующих системных языков (C++, Java) в контексте разработки высоконагруженных сетевых сервисов и облачной инфраструктуры. Основная цель — объединить производительность компилируемых языков с простотой и скоростью разработки динамических языков (Python, JavaScript).

## Ключевые свойства и характеристики

### 1. Компиляция в машинный код
Go компилируется непосредственно в машинный код целевой платформы без необходимости виртуальной машины (как JVM для Java) или интерпретатора. Это обеспечивает:
- Высокую скорость выполнения, сравнимую с C/C++
- Быстрый запуск приложений (нет времени инициализации VM)
- Создание единого самодостаточного исполняемого файла без внешних зависимостей
- Возможность кросс-компиляции (сборка под другие ОС и архитектуры с помощью переменных окружения GOOS и GOARCH)

### 2. Статическая типизация с выводом типов
В Go типы переменных проверяются на этапе компиляции, что предотвращает целый класс ошибок во время выполнения. Однако язык поддерживает вывод типов (type inference) через оператор :=, что снижает многословность кода.

### 3. Сборка мусора (Garbage Collection)
Автоматическое управление памятью освобождает разработчика от необходимости ручного выделения и освобождения памяти. В современных версиях Go (1.8+) сборщик мусора имеет задержку менее 100 микросекунд, что делает его пригодным для real-time систем.

### 4. Встроенная конкурентность
Конкурентность является встроенной в язык, а не реализованной через библиотеки. Примитивы конкурентности:
- **Горутины** (goroutines) — легковесные потоки выполнения, запускаемые ключевым словом go
- **Каналы** (channels) — типизированные средства коммуникации между горутинами, операторы <- и ->
- **Оператор select** — мультиплексирование операций с каналами

Это реализация модели CSP (Communicating Sequential Processes) Хоара.

### 5. Быстрая компиляция
Go спроектирован так, чтобы компилироваться за секунды даже для больших проектов. Это достигается за счет:
- Строгого запрета циклических импортов пакетов
- Простой системы модулей
- Отсутствия сложного препроцессора
- Оптимизированного компилятора

### 6. Простота синтаксиса и языка
Go сознательно лишен многих сложных конструкций:
- Нет классов и наследования (вместо этого композиция и встраивание структур)
- Нет перегрузки операторов
- Нет перегрузки функций
- Нет исключений (try-catch-finally) — вместо них множественные возвращаемые значения с ошибками
- Нет дженериков (до Go 1.18; сейчас есть, но ограниченные)
- Нет заголовочных файлов
- Нет препроцессора
- Нет арифметики указателей

### 7. Строгий форматировщик gofmt
Встроенный инструмент gofmt автоматически форматирует код в едином стиле. Это исключает споры о стиле кодирования и обеспечивает единообразие всей экосистемы Go.

### 8. Интерфейсы неявно (Structural Typing)
В Go тип реализует интерфейс автоматически, если он имеет все требуемые методы. Нет необходимости явно объявлять implements. Это обеспечивает гибкость дизайна и способствует принципу «композиции вместо наследования».

### 9. Многоплатформенность
Go поддерживает кросс-компиляцию для:
- Операционных систем: Linux, Windows, macOS, FreeBSD, OpenBSD, Android, iOS
- Архитектур: amd64, 386, arm, arm64, mips, mips64, ppc64, riscv64, s390x, wasm

### 10. Богатая стандартная библиотека
Стандартная библиотека Go включает мощные пакеты для:
- Сетевого программирования (net, net/http, net/rpc)
- Криптографии (crypto/*)
- Работы с JSON, XML, CSV (encoding/*)
- Шаблонов (text/template, html/template)
- Тестирования (testing)
- Профилирования (runtime/pprof, net/http/pprof)

### 11. Модульная система (Go Modules)
Начиная с Go 1.11 (и по умолчанию с Go 1.16), Go использует модули — систему управления зависимостями с семантическим версионированием, контролем целостности (go.sum) и поддержкой прокси-серверов (GOPROXY).

## Области применения Go
- Высоконагруженные сетевые серверы и микросервисы
- Облачная инфраструктура (Docker, Kubernetes написаны на Go)
- Системное программирование
- CLI-утилиты и DevOps-инструменты
- Blockchain и распределенные системы
- Web-серверы и API-шлюзы
- Обработка данных и потоковая аналитика

## Инструмент командной строки Go

Go поставляется с единым инструментом командной строки `go`, который является центральной точкой входа для всех операций: компиляции, тестирования, управления зависимостями, форматирования и т.д.

### Основные команды и их подробное описание

#### go version
Выводит версию установленного Go, операционную систему и архитектуру.
Пример: go version go1.21.5 linux/amd64

#### go env
Выводит или изменяет переменные окружения Go. Ключевые переменные:
- GOROOT — путь к установке Go (где находятся стандартная библиотека и компилятор)
- GOPATH — рабочее пространство (по умолчанию ~/go), содержит bin, pkg, src
- GOBIN — директория для установки исполняемых файлов при go install
- GOOS — целевая операционная система для сборки
- GOARCH — целевая архитектура процессора
- GO111MODULE — включает/выключает модули (on/off/auto)
- GOPROXY — URL прокси-сервера для загрузки модулей
- GOSUMDB — сервер проверки контрольных сумм модулей

#### go run [файлы...]
Компилирует и немедленно выполняет программу. Не создает постоянный исполняемый файл. Используется для быстрого тестирования и разработки. Под капотом компилирует во временную директорию и запускает.

#### go build [пакеты...]
Компилирует пакеты и их зависимости, но не устанавливает результат. Для package main создает исполняемый файл в текущей директории. Флаги:
- -o имя — задать имя выходного файла
- -v — выводить имена компилируемых пакетов
- -x — выводить выполняемые команды
- -race — включить детектор гонок (race detector)
- -tags — указать build tags

#### go install [пакеты...]
Компилирует и устанавливает исполняемый файл в $GOBIN или $GOPATH/bin. Для сторонних пакетов — основной способ установки CLI-утилит.

#### go get [пакеты...]
Загружает и устанавливает пакеты вместе с их зависимостями. В режиме модулей обновляет go.mod и go.sum. Флаги:
- -u — обновить пакет и его зависимости до последних версий
- -d — только загрузить, не устанавливать

#### go mod <subcommand>
Управление модулями:
- go mod init [модуль] — инициализировать новый модуль, создает go.mod
- go mod tidy — добавить недостающие и удалить неиспользуемые зависимости
- go mod download — загрузить зависимости в локальный кэш
- go mod vendor — создать директорию vendor/ с копиями зависимостей
- go mod verify — проверить целостность зависимостей по go.sum
- go mod graph — вывести граф зависимостей
- go mod why — объяснить, зачем нужна зависимость

#### go test [пакеты...]
Запускает тесты. Ищет файлы *_test.go и выполняет функции Test*, Benchmark*, Fuzz*, Example*. Флаги:
- -v — подробный вывод (verbose)
- -run — запустить только тесты, соответствующие регулярному выражению
- -bench — запустить бенчмарки
- -cover — показать покрытие кода тестами
- -race — включить детектор гонок

#### go fmt [пакеты...]
Форматирует исходный код согласно стандартным правилам gofmt. Может применяться к файлам, пакетам или директориям. Рекомендуется запускать перед коммитом.

#### go vet [пакеты...]
Статический анализатор кода, проверяющий подозрительные конструкции, которые компилятор пропускает, но которые могут быть ошибками:
- Неправильное использование Printf с несовпадающими типами
- Недостижимый код
- Утечки горутин
- Неправильное использование atomic
- Race conditions в тегах структур

#### go doc [пакет/символ]
Генерация и просмотр документации. Показывает документацию к пакетам, типам, функциям, методам. Может запускать локальный сервер документации (godoc -http=:8080).

#### go clean [пакеты...]
Удаляет объектные файлы, кэшированные пакеты и исполняемые файлы, созданные go build, go test и т.д.

#### go generate [пакеты...]
Запускает команды, указанные в директивах //go:generate в исходном коде. Используется для генерации кода (например, protobuf, mock-объектов, SQL-запросов).

#### go list [пакеты...]
Выводит информацию о пакетах в JSON или другом формате. Используется для интеграции с инструментами и скриптами.

#### go work <subcommand>
Управление workspace (Go 1.18+): объединение нескольких модулей для совместной разработки.

### Кросс-компиляция
Пример сборки под разные платформы:
  GOOS=windows GOARCH=amd64 go build -o app.exe
  GOOS=linux GOARCH=arm64 go build -o app-linux
  GOOS=darwin GOARCH=amd64 go build -o app-mac

## Эволюция структуры проектов в Go

### Эра GOPATH (до Go 1.11)
Все проекты располагались внутри $GOPATH/src. Импорты указывали путь относительно GOPATH/src. Это было неудобно для версионирования и изоляции проектов.

### Эра Go Modules (Go 1.11+)
Современная система управления зависимостями. Проект может находиться в любой директории. Каждый модуль имеет уникальный путь (обычно URL репозитория).

## Файлы конфигурации проекта

### go.mod
Главный файл модуля. Создается командой go mod init. Содержит:
- Директиву module с путем модуля
- Директиву go с минимальной версией Go
- Директивы require, replace, exclude для зависимостей
- Директиву retract для отката версий

Пример go.mod:
  module github.com/company/project
  go 1.21
  require (
      github.com/gorilla/mux v1.8.1
      github.com/lib/pq v1.10.9
  )
  require golang.org/x/crypto v0.17.0 // indirect

### go.sum
Файл контрольных сумм. Содержит криптографические хеши (SHA-256) всех зависимостей и их транзитивных зависимостей. Обеспечивает воспроизводимость сборки и защиту от подмены зависимостей. Не редактируется вручную.

### go.work (Go 1.18+)
Файл workspace для работы с несколькими модулями одновременно. Полезен при разработке микросервисов или библиотек с зависимостями между собой.

Пример go.work:
  go 1.21
  use (
      ./backend
      ./frontend
      ./shared
  )
  replace github.com/company/shared => ./shared

### .go-version
Файл, указывающий требуемую версию Go (используется менеджерами версий типа gvm, asdf).

### Makefile / Taskfile.yml
Скрипты сборки, тестирования, развертывания.

### Dockerfile, docker-compose.yml
Контейнеризация приложения.

## Стандартная структура директорий проекта

### /cmd
Содержит точки входа (main-пакеты) для различных исполняемых файлов.
Структура: /cmd/server/main.go, /cmd/cli/main.go, /cmd/worker/main.go
Каждая поддиректория — отдельное приложение.

### /internal
Приватные пакеты, которые не могут быть импортированы другими модулями (защита компилятора Go). Содержит бизнес-логику, привязанную к конкретному приложению.
Поддиректории:
- /internal/handlers — HTTP-обработчики
- /internal/models — доменные модели/сущности
- /internal/repository — доступ к данным
- /internal/service — бизнес-логика
- /internal/config — конфигурация
- /internal/middleware — промежуточные обработчики

### /pkg
Публичные библиотеки, которые могут быть импортированы другими проектами. Код, предназначенный для повторного использования.

### /api
Спецификации API: OpenAPI/Swagger файлы, Protocol Buffers (.proto), GraphQL схемы.

### /web
Статические файлы фронтенда: HTML, CSS, JavaScript, изображения.

### /configs
Файлы конфигурации: YAML, JSON, TOML, конфиги Kubernetes, CI/CD.

### /scripts
Скрипты сборки, миграции баз данных, развертывания.

### /tests
Интеграционные и end-to-end тесты, тестовые данные, фикстуры.

### /docs
Документация проекта.

### /assets
Другие ресурсы, встраиваемые в бинарник (go:embed).

### /vendor (опционально)
Копии зависимостей для воспроизводимости сборки без внешней сети.

## Переменные окружения

### GOROOT
Путь к установке Go. Содержит компилятор, стандартную библиотеку, инструменты. Обычно /usr/local/go или C:\Go.

### GOPATH
Рабочее пространство. По умолчанию ~/go. Содержит:
- bin — скомпилированные исполняемые файлы
- pkg — скомпилированные пакеты (кэш)
- src — исходный код (в эпоху GOPATH)

### GOBIN
Директория для установки бинарников через go install. По умолчанию $GOPATH/bin.

### GOFLAGS
Флаги, передаваемые go command по умолчанию.

## Пакеты в Go

Пакет (package) — основная единица организации кода в Go. Каждый файл Go начинается с объявления пакета. Файлы в одной директории должны принадлежать одному пакету (исключение: файлы _test.go могут иметь пакет xxx_test).

### Типы пакетов

1. **Исполняемые пакеты (package main)**
   - Содержат функцию main()
   - Компилируются в исполняемые файлы
   - Точка входа в программу

2. **Библиотечные пакеты (package <name>)**
   - Предоставляют функциональность для импорта
   - Не компилируются в исполняемые файлы
   - Имя пакета обычно совпадает с именем директории

### Правила видимости
- Идентификаторы, начинающиеся с заглавной буквы (Exported), видны из других пакетов
- Идентификаторы с прописной буквы (unexported), видны только внутри пакета
- Это касается переменных, функций, методов, типов, полей структур

### Импорт пакетов
- import "fmt" — одиночный импорт
- import ( "fmt"; "net/http" ) — групповой импорт
- import m "math" — импорт с алиасом
- import _ "database/sql/driver" — импорт для side effects (например, регистрация драйвера)
- import . "fmt" — импорт в глобальное пространство имен (не рекомендуется)

### Циклические импорты
Go строго запрещает циклические зависимости между пакетами. Это принудительно упрощает архитектуру и ускоряет компиляцию.

## Точка входа

### Функция main

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
```

Требования к main:
- Пакет должен называться main
- Функция main не имеет параметров и возвращаемых значений
- В программе должна быть ровно одна функция main
- main() неявно вызывается runtime после инициализации всех пакетов

### Порядок инициализации
1. Импортируются все пакеты (рекурсивно)
2. Для каждого пакета вычисляются значения констант и переменных уровня пакета
3. Для каждого пакета вызывается функция init() (если есть)
4. Вызывается main.main()

### Функция init()

- Специальная функция, вызываемая автоматически при инициализации пакета
- Не имеет параметров и возвращаемых значений
- В одном пакете может быть несколько init()-функций (в разных файлах)
- Вызываются в порядке объявления в файлах
- Используется для: регистрации драйверов, инициализации глобальных переменных, проверки конфигурации

Пример:
```go
package database

import "database/sql"

func init() {
    sql.Register("custom", &CustomDriver{})
}
```

## Функции в Go

### Объявление функции
```go
func имя(параметры) (возвращаемые_значения) {
    тело
}
```

### Особенности функций Go
- Функции являются first-class citizens (их можно передавать как аргументы, возвращать, присваивать переменным)
- Поддержка замыканий (closures)
- Поддержка variadic-параметров (переменное число аргументов)
- Именованные возвращаемые значения
- Множественные возвращаемые значения (включая ошибки)
- defer — отложенный вызов функций
- recover — восстановление после panic

### Структура HTTP-приложения
```go
package main

import (
    "net/http"
    "log"
)

func main() {
    // Регистрация обработчиков
    http.HandleFunc("/", homeHandler)
    http.HandleFunc("/api/users", usersHandler)

    // Запуск сервера
    log.Println("Server starting on :8080")
    log.Fatal(http.ListenAndServe(":8080", nil))
}

func homeHandler(w http.ResponseWriter, r *http.Request) {
    w.Write([]byte("Home"))
}

func usersHandler(w http.ResponseWriter, r *http.Request) {
    w.Header().Set("Content-Type", "application/json")
    w.Write([]byte(`{"users":[]}`))
}
```

## Категории типов данных

### Целочисленные типы

| Тип | Размер | Диапазон |
|-----|--------|----------|
| int8 | 1 байт | -128 до 127 |
| int16 | 2 байта | -32768 до 32767 |
| int32 | 4 байта | -2^31 до 2^31-1 |
| int64 | 8 байт | -2^63 до 2^63-1 |
| uint8 | 1 байт | 0 до 255 |
| uint16 | 2 байта | 0 до 65535 |
| uint32 | 4 байта | 0 до 2^32-1 |
| uint64 | 8 байт | 0 до 2^64-1 |
| int | зависит от платформы | 32 или 64 бита |
| uint | зависит от платформы | 32 или 64 бита |
| uintptr | зависит от платформы | целое, достаточное для хранения указателя |

### Вещественные типы
| Тип | Размер | Точность |
|-----|--------|----------|
| float32 | 4 байта | ~7 знаков |
| float64 | 8 байт | ~15 знаков |

### Комплексные типы
| Тип | Размер |
|-----|--------|
| complex64 | 8 байт (float32 + float32) |
| complex128 | 16 байт (float64 + float64) |

### Строковый тип
string — неизменяемая последовательность байт в кодировке UTF-8. Длина строки — количество байт, не символов.

### Булев тип
bool — true или false. Zero value: false.

### Специальные алиасы
- byte = uint8 — используется для работы с сырыми байтами
- rune = int32 — представляет Unicode code point (символ)

## Объявления переменных

### Полная форма (var)
```go
var x int = 10
var y = 20  // вывод типа
var z int   // zero value: 0
```

### Краткая форма (:=)
Доступна только внутри функций. Компилятор выводит тип из правой части.
```go
x := 10
name := "Ivan"
price := 99.99
```

### Множественное объявление
```go
var a, b, c = 1, 2, 3
var (
    name    = "Server"
    port    = 8080
    enabled = true
)
```

### Zero values
В Go переменные всегда инициализируются значением по умолчанию, если не указано иное:
- Числовые: 0
- Булевы: false
- Строки: "" (пустая строка)
- Указатели, слайсы, карты, функции, интерфейсы, каналы: nil

## Константы

### Объявление констант
```go
const Pi = 3.14159
const (
    Monday = iota  // 0
    Tuesday        // 1
    Wednesday      // 2
)
```

### Типы констант
- **Untyped constants** — не имеют фиксированного типа, могут использоваться в любом контексте, где это допустимо
- **Typed constants** — имеют явный тип

```go
const untyped = 42       // untyped integer
const typed int64 = 42   // typed integer
```

### Генератор констант iota
iota — предопределенный идентификатор, представляющий порядковый номер константы в блоке const, начиная с 0.

```go
const (
    _ = iota             // 0, пропускаем
    KB = 1 << (10 * iota) // 1 << 10 = 1024
    MB = 1 << (10 * iota) // 1 << 20 = 1048576
    GB = 1 << (10 * iota) // 1 << 30
)
```

### Правила констант
- Константы вычисляются на этапе компиляции
- Могут быть только числовыми, строковыми или булевыми
- Нельзя объявить константу слайс, карту или структуру
- Арифметические операции с константами выполняются с произвольной точностью

ВОПРОС 6. GO: указатели, работа с указателями
================================================================================

## Концепция указателей

Указатель — это переменная, хранящая адрес памяти другой переменной. В Go указатели являются безопасными — нет арифметики указателей, нет неявного преобразования между указателями разных типов.

### Объявление и использование
```go
var x int = 10
var p *int = &x  // p хранит адрес x
fmt.Println(*p)   // разыменование: 10
*p = 20           // изменение x через указатель
```

### Операторы
- & — оператор взятия адреса (address-of)
- * — оператор разыменования (dereference)
- *T — тип "указатель на T"

### Нулевой указатель
Нулевое значение указателя — nil. Разыменование nil-указателя вызывает panic.

```go
var p *int
fmt.Println(p == nil) // true
// *p = 10 // PANIC: runtime error: invalid memory address or nil pointer dereference
```

### Указатели в функциях
Передача по указателю позволяет функции модифицировать оригинальную переменную:
```go
func increment(val *int) {
    *val++
}
```

### Возврат указателей на локальные переменные
В Go безопасно возвращать указатели на локальные переменные. Компилятор обнаруживает, что переменная "убегает" из стека (escape analysis), и размещает её в куче (heap).

```go
func newInt() *int {
    x := 42
    return &x  // безопасно! x будет в куче
}
```

### Указатели на структуры
```go
type Point struct { X, Y int }
p := &Point{1, 2}
fmt.Println(p.X)  // автоматическое разыменование: (*p).X
```

### new() и make()
- new(T) — выделяет память для типа T, возвращает *T, инициализирует zero value
- make() — используется только для слайсов, карт и каналов, возвращает инициализированный тип (не указатель)

### Сравнение указателей
Указатели можно сравнивать на равенство (== и !=). Два указателя равны, если они указывают на один и тот же объект или оба nil.

### Ограничения Go
- Нет арифметики указателей: p++ запрещено
- Нет преобразования указателей в целые числа (кроме unsafe.Pointer)
- Нет указателей на функции-члены (method pointers) как в C++

ВОПРОС 7. GO: массивы, многомерные массивы, работа с массивами
================================================================================

## Массивы в Go

Массив — это нумерованная последовательность элементов одного типа фиксированной длины. Длина массива является частью его типа: [5]int и [10]int — разные типы.

### Объявление и инициализация
```go
var arr1 [5]int                    // [0, 0, 0, 0, 0]
var arr2 = [5]int{1, 2, 3, 4, 5}  // полная инициализация
var arr3 = [5]int{1, 2}            // [1, 2, 0, 0, 0]
var arr4 = [...]int{1, 2, 3}       // длина выводится: [3]int
var arr5 = [5]int{2: 100, 4: 200}  // [0, 0, 100, 0, 200]
```

### Свойства массивов
- **Значимый тип** — при присваивании и передаче в функцию массив копируется полностью
- **Фиксированная длина** — определяется на этапе компиляции
- **Индексация** — с 0 до len-1
- **Границы проверяются** — выход за границы вызывает panic

### Операции с массивами
```go
arr := [5]int{1, 2, 3, 4, 5}
len(arr)    // длина: 5
cap(arr)    // емкость: 5 (для массивов равна длине)
arr[0]      // доступ по индексу
arr[0] = 10 // присваивание
```

### Обход массива
```go
// Классический цикл
for i := 0; i < len(arr); i++ {
    fmt.Println(arr[i])
}

// Range
for i, v := range arr {
    fmt.Printf("index=%d, value=%d\n", i, v)
}

// Только индексы
for i := range arr { ... }

// Только значения (игнорирование индекса)
for _, v := range arr { ... }
```

### Многомерные массивы
```go
// Двумерный массив
var matrix [3][3]int
matrix[0][0] = 1
matrix[1][1] = 1
matrix[2][2] = 1

// Инициализация
matrix2 := [3][3]int{
    {1, 0, 0},
    {0, 1, 0},
    {0, 0, 1},
}

// Трехмерный массив
cube := [2][2][2]int{
    {{1, 2}, {3, 4}},
    {{5, 6}, {7, 8}},
}

// Обход многомерного массива
for i := 0; i < len(matrix); i++ {
    for j := 0; j < len(matrix[i]); j++ {
        fmt.Print(matrix[i][j], " ")
    }
    fmt.Println()
}
```

### Передача массивов в функции
Поскольку массивы копируются, передача больших массивов неэффективна. Рекомендуется передавать указатели или слайсы:
```go
func process(arr *[5]int) { ... }  // указатель
func process(arr []int) { ... }    // слайс (предпочтительно)
```

### Сравнение массивов
Массивы можно сравнивать на == и !=, если элементы сравнимы. Массивы равны, если все соответствующие элементы равны.

ВОПРОС 8. GO: срезы (slice), работа со срезами
================================================================================

## Концепция срезов (Slice)

Срез — это динамическая структура данных, представляющая собой дескриптор сегмента массива. Внутри срез содержит три компонента:
1. **Указатель** на базовый массив
2. **Длина (length)** — количество элементов в срезе
3. **Емкость (capacity)** — количество элементов от начала среза до конца базового массива

Срез — это ссылочный тип (reference type). При присваивании среза копируется только дескриптор (24 байта на 64-битной системе), а не данные.

### Создание срезов
```go
// Литерал
s1 := []int{1, 2, 3}  // len=3, cap=3

// Функция make
s2 := make([]int, 5)      // len=5, cap=5, zero values
s3 := make([]int, 5, 10)  // len=5, cap=10

// Из массива
arr := [5]int{1, 2, 3, 4, 5}
s4 := arr[1:4]  // [2, 3, 4], ссылается на arr!

// Из другого среза
s5 := s4[1:2]   // [3]
```

### Операции со срезами

#### Добавление элементов (append)
```go
s := []int{1, 2}
s = append(s, 3)          // [1, 2, 3]
s = append(s, 4, 5)       // [1, 2, 3, 4, 5]
s = append(s, anotherSlice...)  // распаковка среза
```

**Важно:** append может переаллоцировать память, если capacity недостаточна. В этом случае создается новый базовый массив, и срез начинает ссылаться на него.

#### Копирование
```go
dst := make([]int, len(src))
n := copy(dst, src)  // n — количество скопированных элементов
```

#### Срез среза (reslicing)
```go
s := []int{0, 1, 2, 3, 4, 5}
s1 := s[2:5]  // [2, 3, 4]
s2 := s[:3]   // [0, 1, 2]
s3 := s[3:]   // [3, 4, 5]
s4 := s[:]    // [0, 1, 2, 3, 4, 5]
```

#### Удаление элементов
```go
// Удаление по индексу i
s = append(s[:i], s[i+1:]...)

// Удаление первого элемента
s = s[1:]

// Удаление последнего элемента
s = s[:len(s)-1]
```

### Двумерные срезы
```go
matrix := make([][]int, 3)  // 3 строки
for i := range matrix {
    matrix[i] = make([]int, 4)  // 4 столбца
}
```

### Подводные камни
- Изменение элементов среза, полученного из массива, изменяет исходный массив
- append внутри функции не изменит длину оригинального среза без возврата
- Нельзя сравнивать срезы на == (кроме сравнения с nil)
- Для сравнения содержимого используйте reflect.DeepEqual или ручное сравнение

ВОПРОС 9. GO: ассоциативные массивы (map), работа с ассоциативными массивами
================================================================================

## Карты (Map) в Go

Map — это встроенная хеш-таблица, реализующая ассоциативный массив. Map хранит пары ключ-значение и обеспечивает среднее время доступа O(1).

### Объявление и создание
```go
// Объявление (nil map)
var m map[string]int

// Создание через make
m = make(map[string]int)
m = make(map[string]int, 100)  // с начальной емкостью

// Литерал
m2 := map[string]int{
    "one": 1,
    "two": 2,
}
```

**Важно:** nil map нельзя заполнять (запись вызовет panic). Нужно обязательно инициализировать через make или литерал.

### Операции
```go
// Запись
m["key"] = 100

// Чтение
val := m["key"]  // если ключ отсутствует, вернется zero value типа

// Проверка существования (comma ok idiom)
val, ok := m["key"]
if ok {
    fmt.Println("Key exists:", val)
} else {
    fmt.Println("Key not found")
}

// Удаление
delete(m, "key")

// Длина
len(m)  // количество пар
```

### Обход карты
```go
for key, value := range m {
    fmt.Println(key, value)
}

// Только ключи
for key := range m { ... }

// Только значения
for _, value := range m { ... }
```

**Важно:** Порядок обхода range по map не определен и случаен! Не полагайтесь на порядок.

### Требования к ключам
Ключом map может быть любой сравнимый тип:
- Разрешено: bool, numeric, string, pointer, channel, interface, struct (если все поля сравнимы), array
- Запрещено: slice, map, function

### Вложенные карты
```go
users := map[string]map[string]string{
    "admin": {
        "name": "Ivan",
        "role": "administrator",
    },
}
```

### Потокобезопасность
Map в Go не потокобезопасен! Для конкурентного доступа используйте:
- sync.RWMutex — ручная синхронизация
- sync.Map — потокобезопасная map (для специфических сценариев, медленнее обычной)

### Производительность
- Время доступа: O(1) среднее, O(n) худшее (при коллизиях)
- Map растет динамически при заполнении
- Для больших карт предварительное выделение через make с capacity улучшает производительность

ВОПРОС 10. GO: пользовательские типы, работа с пользовательскими типами
================================================================================

## Определение пользовательских типов

В Go можно создавать новые типы на основе существующих с помощью ключевого слова type. Это создает абсолютно новый тип с отдельным набором операций.

```go
type UserID uint64
type Temperature float64
type StatusCode int
type Username string
```

## Отличие от алиасов

Пользовательский тип — это новый тип, несовместимый с базовым:
```go
type MyInt int
var a MyInt = 5
var b int = a  // ОШИБКА: несовместимые типы
var c int = int(a)  // нужно явное преобразование
```

Алиас (type T = S) — это просто другое имя для того же типа.

## Методы для пользовательских типов

Go позволяет определять методы для любых именованных типов (не только для структур):

```go
type UserID uint64

func (id UserID) String() string {
    return fmt.Sprintf("UID:%d", id)
}

func (id UserID) IsValid() bool {
    return id > 0
}

type Temperature float64

func (t Temperature) Celsius() float64 {
    return float64(t)
}

func (t Temperature) Fahrenheit() float64 {
    return float64(t)*9/5 + 32
}
```

## Применение пользовательских типов

### Типобезопасность
```go
type UserID int
type OrderID int

func GetUser(id UserID) { ... }
func GetOrder(id OrderID) { ... }

GetUser(UserID(42))      // OK
GetUser(OrderID(42))     // ОШИБКА компиляции!
```

### Предметная область
Пользовательские типы делают код самодокументируемым и предотвращают ошибки смешения различных сущностей.

### Интерфейсы
Пользовательские типы могут реализовывать интерфейсы, определяя методы:
```go
type Stringer interface {
    String() string
}

func (id UserID) String() string { ... }  // UserID реализует Stringer
```

### Ограничения
- Методы можно определять только для типов в том же пакете
- Нельзя определять методы для встроенных типов или типов из других пакетов
- Нельзя определять методы для указателей на встроенные типы

ВОПРОС 11. GO: алиасы, работа алиасами
================================================================================

## Алиасы типов

Алиас (type alias) — это альтернативное имя для существующего типа. В отличие от пользовательского типа, алиас не создает новый тип — это тот же самый тип с другим именем.

### Синтаксис
```go
type MyString = string  // алиас
type MyString2 string   // новый тип (не алиас!)
```

### Встроенные алиасы
Go имеет два предопределенных алиаса:
- byte = uint8
- rune = int32

### Поведение алиасов
```go
type MyString = string
var s MyString = "hello"
var str string = s  // OK! Это один и тот же тип

// С пользовательским типом:
type MyString2 string
var s2 MyString2 = "hello"
// var str2 string = s2  // ОШИБКА: несовместимые типы
```

## Алиасы пакетов

При импорте можно задавать алиасы для пакетов:
```go
import (
    "fmt"
    m "math"           // алиас m
    . "strings"         // импорт в глобальное пространство
    _ "database/sql/driver"  // импорт для side effects
)

func main() {
    fmt.Println(m.Pi)
    fmt.Println(ToUpper("hello"))  // из strings, без префикса
}
```

## Применение алиасов

### Обратная совместимость
Алиасы используются для постепенного переименования типов без поломки существующего кода:
```go
// Старый код использует OldName
// Новый код может использовать NewName
type OldName = NewName
```

### Разрешение конфликтов имен
```go
import (
    "text/template"
    htmltemplate "html/template"
)
```

### Крупномасштабный рефакторинг
В Go 1.9 алиасы были добавлены именно для поддержки постепенного рефакторинга больших кодовых баз.

ВОПРОС 12. GO: структуры, работа со структурами
================================================================================

## Структуры (struct)

Структура — это составной тип, объединяющий ноль или более полей различных типов под одним именем. Структуры являются основой для создания сложных типов данных и объектов в Go.

### Объявление
```go
type User struct {
    ID        int
    Name      string
    Email     string
    IsActive  bool
    CreatedAt time.Time
}
```

### Создание экземпляров
```go
// С именованными полями
u1 := User{ID: 1, Name: "Ivan"}

// Позиционная инициализация (не рекомендуется для больших структур)
u2 := User{2, "Petr", "petr@mail.ru", true, time.Now()}

// Через new (возвращает указатель)
u3 := new(User)  // &User{}, все поля zero values

// Через указатель с именованными полями
u4 := &User{Name: "Anna"}

// Zero value
var u5 User  // {0, "", "", false, 0001-01-01 00:00:00 +0000 UTC}
```

### Доступ к полям
```go
u.Name = "Sergey"
fmt.Println(u.ID, u.Name)
```

## Вложенные структуры (композиция)

Go не поддерживает наследование классов. Вместо этого используется композиция — вложение одних структур в другие.

### Анонимное вложение (promotion)
```go
type User struct {
    ID   int
    Name string
}

type Employee struct {
    User           // анонимное вложение (promoted fields)
    Department string
    Salary     float64
}

e := Employee{
    User:       User{ID: 1, Name: "Ivan"},
    Department: "IT",
    Salary:     50000,
}

// Прямой доступ к promoted fields!
fmt.Println(e.Name)     // "Ivan" — доступ напрямую
fmt.Println(e.User.Name) // "Ivan" — явный доступ
```

### Именованное вложение
```go
type Employee struct {
    Person     User   // именованное вложение
    Department string
}

e := Employee{Person: User{ID: 1, Name: "Ivan"}}
fmt.Println(e.Person.Name)  // только явный доступ
```

## Методы структур

```go
func (u User) GetInfo() string {
    return fmt.Sprintf("%s (ID: %d)", u.Name, u.ID)
}

func (u *User) SetName(name string) {
    u.Name = name  // изменение через указатель
}
```

### Выбор получателя: значение vs указатель
- **Значение (func (u User))** — метод получает копию структуры. Изменения не видны снаружи. Безопасен для конкурентности.
- **Указатель (func (u *User))** — метод получает указатель. Изменения видны снаружи. Не копирует структуру (эффективно для больших структур).

## Теги структур (Struct Tags)

Строковые метаданные, прикрепленные к полям структуры. Используются для:
- Сериализации JSON/XML
- Маппинга баз данных (ORM)
- Валидации
- Флагов командной строки

```go
type Product struct {
    ID          int       `json:"id" db:"product_id"`
    Name        string    `json:"name" validate:"required,max=100"`
    Price       float64   `json:"price,omitempty"`
    Description string    `json:"-"`  // игнорировать при сериализации
}
```

Теги читаются через рефлексию (reflect.StructTag).

## Сравнение структур

Структуры можно сравнивать на == и !=, если все их поля сравнимы. Нельзя сравнивать структуры, содержащие слайсы, карты или функции.

## Строки в Go

Строка в Go — это неизменяемая (immutable) последовательность байт. По умолчанию строки хранятся в кодировке UTF-8, но сам тип string не гарантирует валидности UTF-8 — это просто последовательность байт.

### Свойства строк
- **Неизменяемость** — нельзя изменить отдельный байт: s[0] = 'x' запрещено
- **Индексация по байтам** — s[i] возвращает байт (uint8), не символ!
- **Длина len(s)** — количество байт, не символов
- **Zero value** — пустая строка ""

### Создание строк
```go
s1 := "Hello, World!"
s2 := `Raw string
with backslashes`  // raw string literal
s3 := string([]byte{72, 101, 108, 108, 111})  // из байтов
s4 := string(rune(1055))  // "П" — из руны
```

### Обход строки

#### По байтам (неправильно для Unicode!)
```go
for i := 0; i < len(s); i++ {
    fmt.Printf("%d: %x\n", i, s[i])
}
```

#### По рунам (правильно)
```go
for i, r := range s {
    fmt.Printf("%d: %c (U+%04X)\n", i, r, r)
}
```

При range по строке Go автоматически декодирует UTF-8. i — байтовый индекс, r — руна (int32).

### Конкатенация
```go
s := "Hello" + " " + "World"
```

Для множественной конкатенации используйте strings.Builder (эффективнее):
```go
var b strings.Builder
for _, s := range words {
    b.WriteString(s)
}
result := b.String()
```

### Пакет strings
```go
strings.Contains(s, "sub")       // содержит подстроку
strings.HasPrefix(s, "pre")      // начинается с
strings.HasSuffix(s, "suf")      // заканчивается на
strings.ToUpper(s) / ToLower(s)  // регистр
strings.TrimSpace(s)             // убрать пробелы
strings.Trim(s, "cutset")        // убрать символы
strings.Split("a,b,c", ",")      // разделить
strings.Join([]string{"a","b"}, ",")  // соединить
strings.Replace(s, "old", "new", n)  // заменить n вхождений
strings.Repeat(s, 3)             // повторить
strings.Index(s, "sub")          // индекс подстроки
```

### Пакет strconv
```go

strconv.Itoa(42)             // int -> string
strconv.ParseInt("42", 10, 64)  // string -> int64

```

### Руны (runes)

Руна — это Unicode code point, тип rune = int32.
```go
r := 'П'  // руна (в одинарных кавычках)
runes := []rune(s)  // строка -> слайс рун
s = string(runes)   // слайс рун -> строка
```

### Пакет unicode/utf8

## Арифметические операторы

| Оператор | Описание |
|----------|----------|
| + | Сложение |
| - | Вычитание |
| * | Умножение |
| / | Деление (целочисленное для int, вещественное для float) |
| % | Остаток от деления |
| ++ | Инкремент (только постфиксный) |

## Побитовые операторы

| Оператор | Описание |
|----------|----------|
| & | Побитовое И |
| \| | Побитовое ИЛИ |
| ^ | Побитовое исключающее ИЛИ (XOR) |
| &^ | И НЕ (AND NOT, битовая очистка) |
| << | Побитовый сдвиг влево |
| >> | Побитовый сдвиг вправо |

```go
x := 0b1010  // 10

x & y   // 0b1000 = 8
x | y   // 0b1110 = 14
x ^ y   // 0b0110 = 6
x &^ y  // 0b0010 = 2 (биты y сбрасываются в x)
x << 2  // 0b101000 = 40
x >> 1  // 0b101 = 5
```

## Операторы сравнения

| Оператор | Описание |
|----------|----------|
| == | Равно |
| != | Не равно |
| < | Меньше |
| > | Больше |
| <= | Меньше или равно |
| >= | Больше или равно |

Операторы == и != применимы к типам, которые можно сравнивать. Слайсы, карты, функции можно сравнивать только с nil.

## Логические операторы

| Оператор | Описание |
|----------|----------|
| && | Логическое И (short-circuit) |
| \|\| | Логическое ИЛИ (short-circuit) |
| ! | Логическое НЕ |

**Short-circuit evaluation:** В выражении `a && b`, если a == false, b не вычисляется. В `a || b`, если a == true, b не вычисляется.
пример
x := 5  // 0101 в двоичной
y := 3  // 0011

fmt.Println(x & y)   // 1 (0001) - AND (И)
fmt.Println(x | y)   // 7 (0111) - OR (ИЛИ)
fmt.Println(x ^ y)   // 6 (0110) - XOR (исключающее ИЛИ)
fmt.Println(x &^ y)  // 4 (0100) - AND NOT (сбросить биты)

// Сдвиги
fmt.Println(x << 1)  // 10 (1010) - сдвиг влево (умножение на 2)
fmt.Println(x >> 1)  // 2  (0010) - сдвиг вправо (деление на 2)

| Оператор | Описание |
|----------|----------|
| + | Унарный плюс |
| - | Унарный минус (арифметическое отрицание) |
| ! | Логическое отрицание |
| ^ | Побитовое отрицание (НЕ) |
| * | Разыменование указателя |
| & | Взятие адреса |
| <- | Получение из канала |

- Нет тернарного оператора ?:
- Нет неявного преобразования типов: int32 нельзя присвоить int64 без явного преобразования

- Оператор &^ (AND NOT) уникален для Go
- Порядок вычисления операндов строго определен слева направо

## Условный оператор if

### Базовый синтаксис
```go
if условие {
    // тело
} else if другое_условие {
    // тело
} else {
    // тело
}
```

### if с инициализацией (short statement)
```go
if err := doSomething(); err != nil {
    return err
}
// err недоступна здесь — область видимости ограничена if
```

Это идиоматично для Go и позволяет ограничить область видимости переменных.

### Особенности if в Go
- Условие не требует скобок
- Тело всегда в фигурных скобках (даже для одной строки)
- else if записывается без переноса строки перед else
- Переменные, объявленные в инициализационной части, видны только внутри if/else

## Оператор switch

### Классический switch
```go
switch выражение {
case значение1:
    // тело
    // break не нужен — он неявный!
case значение2, значение3:
    // несколько значений
    fallthrough  // принудительное проваливание (редко используется)
default:
    // по умолчанию
}
```

### Switch без выражения (tagless switch)
Эквивалент if-else if цепочки:
```go
switch {
case x > 100:
    fmt.Println("big")
case x > 0:
    fmt.Println("positive")
default:
    fmt.Println("zero or negative")
}
```

### Type switch
Проверка динамического типа interface{}:
```go
var i interface{} = 42

switch v := i.(type) {
case int:
    fmt.Println("int:", v)
case string:
    fmt.Println("string:", v)
case float64:
    fmt.Println("float64:", v)
default:
    fmt.Printf("unknown: %T\n", v)
}
```

### Особенности switch в Go
- **Автоматический break** — после выполнения case управление передается за switch. Это предотвращает ошибки проваливания, характерные для C/C++.
- **fallthrough** — явная директива для проваливания в следующий case
- **case может содержать список значений** — case 1, 2, 3:
- **Выражения case вычисляются сверху вниз** — первый совпавший выполняется
- **Порядок case важен** — если есть overlapping условия

## Единственный цикл в Go — for

В Go нет while, do-while, foreach — только for, но с несколькими вариантами использования.

### Классический for (C-style)
```go
for i := 0; i < 10; i++ {
    fmt.Println(i)
}
```

### Условный for (аналог while)
```go
for condition {
    // тело
}

// Пример
for i < 100 {
    i *= 2
}
```

### Бесконечный цикл
```go
for {
    // тело
    if shouldStop {
        break
    }
}
```

### Range-based for
```go
// Слайс или массив
for i, v := range slice {
    // i — индекс, v — копия значения
}

// Только индекс
for i := range slice { ... }

// Только значение (игнорирование индекса)
for _, v := range slice { ... }

// Строка — итерация по рунам
for i, r := range "Go语言" {
    // i — байтовый индекс, r — руна (int32)
}

// Map
for k, v := range m { ... }

// Канал
for v := range ch { ... }  // чтение до закрытия канала
```

### Управление циклами

#### break
Прерывает выполнение цикла. Может использоваться с меткой для выхода из вложенных циклов:
```go
Outer:
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        if i*j > 4 {
            break Outer  // выход из обоих циклов
        }
    }
}
```

#### continue
Переходит к следующей итерации. Также поддерживает метки:
```go
Outer:
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        if j == 1 {
            continue Outer  // переход к следующей итерации внешнего цикла
        }
    }
}
```

### Особенности range
- При range по слайсу/массиву v — копия элемента. Изменение v не влияет на оригинал.
- При range по map порядок итерации случайный!
- При range по каналу цикл завершается при закрытии канала.
- При range по строке индекс — байтовый, а значение — руна.

### for с инициализацией нескольких переменных
```go
for i, j := 0, len(arr)-1; i < j; i, j = i+1, j-1 {
    arr[i], arr[j] = arr[j], arr[i]  // reverse array
}
```

```go
func divide(a, b float64) (float64, error) {
```

```go
}

sum(1, 2, 3)
sum(slice...)  // распаковка слайса
```

### Функции как значения (first-class citizens)

### Замыкания (closures)
```go
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

c := counter()
```

### Отложенные вызовы (defer)

defer откладывает выполнение функции до возврата из окружающей функции. Вызовы defer помещаются в стек и выполняются в порядке LIFO (последний defer — первый выполнится).

```go
func readFile(filename string) error {
    f, err := os.Open(filename)
    if err != nil {
        return err
    }
    defer f.Close()  // гарантированно закроется при выходе

    // Работа с файлом...
    return nil
}
```

**Идиома:** defer используется для освобождения ресурсов (закрытие файлов, освобождение блокировок, завершение транзакций).

**Особенности defer:**
- Аргументы defer вычисляются немедленно, но вызов откладывается
- defer может изменить именованные возвращаемые значения
- Накладные расходы defer минимальны (в Go 1.14+ снижены на 80%)

### Восстановление после паники (recover)

```go
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered from panic:", r)
        }
    }()

    panic("something went wrong")
    // Эта строка не выполнится
}
```

recover перехватывает panic только внутри defer-функции. Это используется для:
- Отказоустойчивости серверов (не падаем из-за одного запроса)
- Очистки ресурсов при аварийном завершении

### panic
panic немедленно прерывает выполнение функции и начинает раскрутку стека (unwinding), вызывая defer-функции. Если recover не перехватит panic, программа завершится с ошибкой.

## Горутины (Goroutines)

Горутина — это легковесный поток выполнения, управляемый Go runtime, а не операционной системой. Горутины являются ключевой концепцией конкурентности в Go.

### Характеристики горутин
- **Легковесность** — начальный стек 2KB (вместо 1-8MB у OS thread), динамически растет и сжимается
- **Управляемость** — Go runtime планирует горутины на меньшем количестве OS threads (M:N scheduling)
- **Быстрый запуск** — создание горутины занимает микросекунды
- **Масштабируемость** — можно создавать сотни тысяч горутин

### Запуск горутины
```go
go function()      // запуск функции в горутине
go func() { ... }() // запуск анонимной функции
```

Пример:
```go
func say(s string) {
    for i := 0; i < 5; i++ {
        time.Sleep(100 * time.Millisecond)
        fmt.Println(s)
    }
}

func main() {
    go say("world")  // запуск в горутине
    say("hello")     // выполнение в главной горутине
}
```

## Каналы (Channels)

Канал — это типизированная средство коммуникации между горутинами. Каналы обеспечивают синхронизацию и передачу данных.

### Создание каналов
```go
ch := make(chan int)      // небуферизованный канал (синхронный)
ch := make(chan int, 10)  // буферизованный канал (емкость 10)
```

### Операции с каналами
```go
ch <- v      // отправка значения в канал (блокирует, если канал полон или небуферизован)
v := <-ch    // получение значения из канала (блокирует, если канал пуст)
v, ok := <-ch // ok=false если канал закрыт и пуст
close(ch)    // закрытие канала (отправлять больше нельзя, получать можно)
```

### Небуферизованные каналы
Синхронны: отправка блокируется до тех пор, пока другая горутина не получит значение. Обеспечивают строгую синхронизацию (rendezvous).

### Буферизованные каналы
Асинхны до заполнения буфера. Отправка блокируется только при заполнении буфера.

### Оператор select

select мультиплексирует операции с каналами:
```go
select {
case v1 := <-ch1:
    fmt.Println("from ch1:", v1)
case v2 := <-ch2:
    fmt.Println("from ch2:", v2)
case ch3 <- 100:
    fmt.Println("sent to ch3")
default:
    fmt.Println("no channel ready")
}
```

- Если несколько case готовы, выбирается случайный (для предотвращения голодания)
- default выполняется, если ни один case не готов (неблокирующий select)
- select без default блокирует до готовности одного из case

## Синхронизация

### sync.WaitGroup
Ожидание завершения группы горутин:
```go
var wg sync.WaitGroup
for i := 0; i < 3; i++ {
    wg.Add(1)
    go func(id int) {
        defer wg.Done()
        fmt.Println("Worker", id)
    }(i)
}
wg.Wait()  // блокирует до завершения всех
```

### sync.Mutex / sync.RWMutex
Взаимное исключение:
```go
var mu sync.Mutex
var counter int

func increment() {
    mu.Lock()
    defer mu.Unlock()
    counter++
}
```

RWMutex позволяет множественное чтение или однократную запись.

### context.Context
Управление жизненным циклом и отмена операций:
```go
ctx, cancel := context.WithCancel(context.Background())
defer cancel()

go func(ctx context.Context) {
    select {
    case <-ctx.Done():
        return  // отмена
    case <-time.After(5 * time.Second):
        fmt.Println("Done")
    }
}(ctx)

cancel()  // отмена всех дочерних горутин
```

Типы контекстов:

## Архитектура database/sql

Пакет database/sql предоставляет универсальный интерфейс для работы с SQL-базами данных. Он реализует паттерн "database driver" — абстракцию, позволяющую работать с различными СУБД через единый API.

### Ключевые компоненты
- **DB** — пул соединений с базой данных
- **Conn** — отдельное соединение (редко используется напрямую)
- **Stmt** — подготовленный запрос (prepared statement)
- **Tx** — транзакция
- **Rows** — результат запроса, возвращающего множество строк
- **Row** — результат запроса, возвращающего одну строку
- **Result** — результат выполнения INSERT/UPDATE/DELETE

### Драйверы
Пакет database/sql требует драйвер для конкретной СУБД. Драйверы регистрируются через side-effect импорт:
- SQLite: github.com/mattn/go-sqlite3
- PostgreSQL: github.com/lib/pq или github.com/jackc/pgx
- MySQL: github.com/go-sql-driver/mysql
- SQL Server: github.com/denisenkom/go-mssqldb

## Подключение

```go
import (
    "database/sql"
    _ "github.com/lib/pq"
)

// DSN (Data Source Name) для PostgreSQL
dsn := "host=localhost port=5432 user=postgres password=secret dbname=mydb sslmode=disable"
db, err := sql.Open("postgres", dsn)
if err != nil {
    log.Fatal(err)
}
defer db.Close()

// Проверка соединения
err = db.Ping()
```

**Важно:** sql.Open не создает соединение сразу! Он только подготавливает пул. Первое реальное соединение устанавливается при первом запросе или при вызове Ping().

## Пакет net/http

Пакет net/http реализует HTTP-клиент и сервер в стандартной библиотеке Go. Он предоставляет минималистичный, но полнофункциональный API для создания веб-сервисов.

### Ключевые типы
- **Handler** — интерфейс с методом ServeHTTP(ResponseWriter, *Request)
- **HandlerFunc** — адаптер, позволяющий использовать обычные функции как Handler
- **ServeMux** — мультиплексор запросов (маршрутизатор)
- **Server** — HTTP-сервер с настраиваемыми параметрами
- **ResponseWriter** — интерфейс для записи ответа
- **Request** — структура, представляющая HTTP-запрос

## Параллельная обработка запросов

Ключевая особенность net/http.Server: каждый входящий HTTP-запрос обрабатывается в **своей собственной горутине**.

```go
package main

import (
    "fmt"
    "net/http"
    "time"
)

func handler(w http.ResponseWriter, r *http.Request) {
    // Этот код выполняется в отдельной горутине для КАЖДОГО запроса
    fmt.Printf("[%s] Started: %s %s from %s\n",
        time.Now().Format("15:04:05"),
        r.Method,
        r.URL.Path,
        r.RemoteAddr,
    )

    // Имитация долгой обработки (2 секунды)
    time.Sleep(2 * time.Second)

    fmt.Fprintf(w, "Processed at %s\n", time.Now().Format(time.RFC3339))

    fmt.Printf("[%s] Finished: %s\n",
        time.Now().Format("15:04:05"),
        r.URL.Path,
    )
}

func main() {
    http.HandleFunc("/", handler)

    fmt.Println("Server starting on :8080")
    fmt.Println("Each request is handled in its own goroutine!")

    // ListenAndServe блокирует главную горутину
    // Но каждый accept() создает новую горутину для обработки
    if err := http.ListenAndServe(":8080", nil); err != nil {
        fmt.Println("Server error:", err)
    }
}
```

### Как это работает внутри
1. Server.Accept() принимает TCP-соединение
2. Для каждого соединения создается новая горутина
3. Внутри горутины происходит HTTP-парсинг и вызов Handler
4. Главная горутина продолжает слушать новые соединения

### Проверка параллельности
```bash
# Запустите три запроса одновременно
# Все три начнутся сразу и завершатся примерно через 2 секунды
# (а не через 6 секунд, как при последовательной обработке)
curl http://localhost:8080/req1 &
curl http://localhost:8080/req2 &
curl http://localhost:8080/req3 &
wait
```

## Пакет fmt

fmt реализует форматированный ввод-вывод, аналогичный C-printf/scanf.

### Функции вывода
- fmt.Print — вывод без форматирования, без перевода строки
- fmt.Println — вывод с пробелами между аргументами и переводом строки
- fmt.Printf — форматированный вывод
- fmt.Fprintf(w, ...) — вывод в io.Writer
- fmt.Sprintf(...) — форматирование в строку

### Спецификаторы формата
- %v — значение по умолчанию
- %+v — значение с именами полей структуры
- %T — тип значения
- %t — булево

- %s — строка

## Маршрутизация в net/http

### ServeMux (DefaultServeMux)
ServeMux — мультиплексор HTTP-запросов. Он сопоставляет URL-пути с обработчиками.

```go
http.HandleFunc("/", homeHandler)       // точное совпадение или префикс
http.HandleFunc("/about", aboutHandler) // точное совпадение
http.HandleFunc("/api/", apiHandler)    // префикс /api/ и все подпути
```

**Правила маршрутизации ServeMux:**
- Проверяется точное совпадение пути
- Если точного нет — ищется ближайший префикс (родительский путь с /)
- / регистрирует корневой обработчик (fallback)
- Пути автоматически очищаются (редирект /../ на /)

### Пользовательский ServeMux
```go
mux := http.NewServeMux()
mux.HandleFunc("/", homeHandler)
mux.HandleFunc("/api/", apiHandler)

http.ListenAndServe(":8080", mux)
```

### Handler vs HandlerFunc
- Handler — интерфейс с методом ServeHTTP
- HandlerFunc — тип функции, адаптирующий обычные функции к Handler

```go
// HandlerFunc
http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
    w.Write([]byte("Hello"))
})

// Handler (интерфейс)
type MyHandler struct{}
func (h MyHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    w.Write([]byte("Hello from Handler"))
}
http.Handle("/custom", MyHandler{})
```

## Работа с JSON

Пакет encoding/json реализует кодирование и декодирование JSON.

### Ключевые функции
- json.Marshal(v) — сериализация в []byte
- json.Unmarshal(data, &v) — десериализация
- json.NewEncoder(w).Encode(v) — сериализация в io.Writer
- json.NewDecoder(r).Decode(&v) — десериализация из io.Reader

### Пример сервера
```go
package main

import (
    "encoding/json"
    "io"
    "log"
    "net/http"
)

type CreateUserRequest struct {
    Name  string `json:"name"`
    Email string `json:"email"`
    Age   int    `json:"age"`
}

type CreateUserResponse struct {
    ID     int    `json:"id"`
    Status string `json:"status"`
}

func createUserHandler(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
        return
    }

    // Проверка Content-Type
    if r.Header.Get("Content-Type") != "application/json" {
        http.Error(w, "Content-Type must be application/json", http.StatusUnsupportedMediaType)
        return
    }

    // Чтение тела
    body, err := io.ReadAll(r.Body)
    if err != nil {
        http.Error(w, "Error reading body", http.StatusBadRequest)
        return
    }
    defer r.Body.Close()

    log.Printf("Received body: %s", string(body))

    // Парсинг JSON
    var req CreateUserRequest
    if err := json.Unmarshal(body, &req); err != nil {
        http.Error(w, "Invalid JSON: "+err.Error(), http.StatusBadRequest)
        return
    }

    // Валидация
    if req.Name == "" || req.Email == "" {
        http.Error(w, "Name and Email are required", http.StatusBadRequest)
        return
    }

    // Логика создания...
    resp := CreateUserResponse{
        ID:     42,
        Status: "created",
    }

    // Формирование ответа
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusCreated)

    if err := json.NewEncoder(w).Encode(resp); err != nil {
        log.Printf("Error encoding response: %v", err)
    }

    log.Printf("Created user: %+v", req)
}

func main() {
    http.HandleFunc("/users", createUserHandler)
    log.Println("Server on :8080")
    log.Fatal(http.ListenAndServe(":8080", nil))
}
```

## Gorilla Mux

Gorilla Mux — мощный маршрутизатор HTTP-запросов и диспетчер для Go. Расширяет стандартный net/http, добавляя:
- Сопоставление с переменными пути (path variables)
- Сопоставление с поддоменами
- Ограничения запросов (схема, хост, заголовки, query-параметры, методы)
- Вложенные маршрутизаторы (subrouters)
- Строгое сопоставление путей (slash, trailing slash)

### Установка
```bash
go get -u github.com/gorilla/mux
```

## Маршрутизация

```go
package main

import (
    "encoding/json"
    "log"
    "net/http"

    "github.com/gorilla/mux"
)

type User struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
}

func getUsers(w http.ResponseWriter, r *http.Request) {
    users := []User{{1, "Ivan"}, {2, "Petr"}}
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(users)
}

func createUser(w http.ResponseWriter, r *http.Request) {
    var u User
    if err := json.NewDecoder(r.Body).Decode(&u); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusCreated)
    json.NewEncoder(w).Encode(u)
}

func main() {
    r := mux.NewRouter()

    // Строгая маршрутизация с методами
    r.HandleFunc("/users", getUsers).Methods("GET")
    r.HandleFunc("/users", createUser).Methods("POST")

    // Middleware
    r.Use(loggingMiddleware)
    r.Use(jsonContentTypeMiddleware)

    log.Println("Server on :8080")
    log.Fatal(http.ListenAndServe(":8080", r))
}

func loggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        log.Printf("%s %s %s", r.RemoteAddr, r.Method, r.URL.Path)
        next.ServeHTTP(w, r)
    })
}

func jsonContentTypeMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Content-Type", "application/json")
        next.ServeHTTP(w, r)
    })
}
```

## Обработка application/x-www-form-urlencoded

Этот формат используется при отправке HTML-форм методом POST. Данные передаются в теле запроса в виде: `key1=value1&key2=value2`.

### Парсинг form-urlencoded
```go
func loginHandler(w http.ResponseWriter, r *http.Request) {
    // ParseForm парсит тело запроса (для POST/PUT/PATCH) и query-параметры URL
    if err := r.ParseForm(); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }

    // Доступ к данным
    username := r.FormValue("username")  // из тела или query (приоритет: тело)
    password := r.PostFormValue("password")  // только из тела

    // Проверка наличия
    if r.Form.Has("remember") {  // Go 1.17+
        // checkbox установлен
    }

    fmt.Fprintf(w, "Username: %s, Password: %s", username, password)
}
```

### Различие FormValue и PostFormValue
- r.FormValue — ищет в r.PostForm (тело), затем в r.Form (query-параметры)
- r.PostFormValue — только в r.PostForm (тело запроса)

### Пример сервера
```go
package main

import (
    "fmt"
    "net/http"

    "github.com/gorilla/mux"
)

func loginHandler(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
        return
    }

    if err := r.ParseForm(); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }

    username := r.PostFormValue("username")
    password := r.PostFormValue("password")

    // Простая валидация
    if username == "" || password == "" {
        http.Error(w, "Username and password required", http.StatusBadRequest)
        return
    }

    fmt.Fprintf(w, "Login successful for user: %s", username)
}

func getUserHandler(w http.ResponseWriter, r *http.Request) {
    vars := mux.Vars(r)
    id := vars["id"]
    fmt.Fprintf(w, "User ID: %s", id)
}

func articleHandler(w http.ResponseWriter, r *http.Request) {
    vars := mux.Vars(r)
    fmt.Fprintf(w, "Category: %s, ID: %s", vars["category"], vars["id"])

    // Получение URL по имени
    route := mux.CurrentRoute(r)
    url, _ := route.URL("category", "tech", "id", "42")
    fmt.Fprintf(w, "\nURL: %s", url.String())
}

func main() {
    r := mux.NewRouter()

    // Шаблоны маршрутов
    r.HandleFunc("/login", loginHandler).Methods("POST")
    r.HandleFunc("/users/{id:[0-9]+}", getUserHandler).Methods("GET")
    r.HandleFunc("/articles/{category}/{id:[0-9]+}", articleHandler).
        Methods("GET").
        Name("article")

    http.ListenAndServe(":8080", r)
}
```

## Комбинированная обработка

```go
package main

import (
    "encoding/json"
    "log"
    "net/http"

    "github.com/gorilla/mux"
)

type Product struct {
    SKU   string  `json:"sku"`
    Name  string  `json:"name"`

func createProduct(w http.ResponseWriter, r *http.Request) {
    // Проверка Content-Type
    if r.Header.Get("Content-Type") != "application/json" {
        http.Error(w, "Content-Type must be application/json", http.StatusUnsupportedMediaType)
        return
    }

    if p.SKU == "" || p.Price <= 0 {
        http.Error(w, "Invalid data: SKU required, Price must be positive", http.StatusBadRequest)
        return
    }

    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusCreated)
    json.NewEncoder(w).Encode(map[string]interface{}{
        "sku":    p.SKU,
        "status": "created",
        "price":  p.Price,
    })
}

func getProduct(w http.ResponseWriter, r *http.Request) {
    vars := mux.Vars(r)
    sku := vars["sku"]

    // Симуляция поиска
    product := Product{SKU: sku, Name: "Sample Product", Price: 99.99}

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(product)
}

func main() {
    r := mux.NewRouter()

    // Subrouter для API
    api := r.PathPrefix("/api/v1").Subrouter()

    // Маршрут с ограничением по методу и заголовку
    api.HandleFunc("/products", createProduct).
        Methods("POST").
        Headers("Content-Type", "application/json")

    // Маршрут с шаблоном переменной
    api.HandleFunc("/products/{sku}", getProduct).
        Methods("GET")

    // Middleware
    r.Use(loggingMiddleware)

    log.Println("Server on :8080")
    log.Fatal(http.ListenAndServe(":8080", r))
}

func loggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        log.Printf("[%s] %s %s", r.Method, r.URL.Path, r.RemoteAddr)
        next.ServeHTTP(w, r)
    })
}
```

```go
package main

import (
    "fmt"
    "net/http"
    "strconv"

    "github.com/gorilla/mux"
)

func getUser(w http.ResponseWriter, r *http.Request) {
    vars := mux.Vars(r)
    idStr := vars["id"]

    // Валидация и преобразование
    id, err := strconv.Atoi(idStr)
    if err != nil {
        http.Error(w, "Invalid user ID: must be integer", http.StatusBadRequest)
        return
    }

    if id <= 0 {
        http.Error(w, "Invalid user ID: must be positive", http.StatusBadRequest)
        return
    }

    fmt.Fprintf(w, "User ID: %d", id)
}

func getUserPost(w http.ResponseWriter, r *http.Request) {
    vars := mux.Vars(r)
    userID := vars["id"]
    postID := vars["post_id"]

    fmt.Fprintf(w, "User: %s, Post: %s", userID, postID)
}

func getFile(w http.ResponseWriter, r *http.Request) {
    path := mux.Vars(r)["path"]

    // Безопасность: предотвращение directory traversal
    // filepath.Clean нормализует путь
    safePath := filepath.Clean(path)
    if strings.Contains(safePath, "..") {
        http.Error(w, "Invalid path", http.StatusBadRequest)
        return
    }

    fmt.Fprintf(w, "File path: %s", safePath)
}

    r := mux.NewRouter()

    // Простой параметр
    r.HandleFunc("/users/{id}", getUser).Methods("GET")

    // Несколько параметров
    r.HandleFunc("/users/{id}/posts/{post_id}", getUserPost).Methods("GET")

    // Параметр с регулярным выражением
```

## Query-параметры (Query String Parameters)

Query-параметры передаются в URL после знака ? в формате `key=value&key2=value2`. Они используются для фильтрации, сортировки, пагинации, поиска.

### Получение query-параметров
```go
func handler(w http.ResponseWriter, r *http.Request) {
    query := r.URL.Query()  // url.Values (map[string][]string)

    // Получение значения (пустая строка, если отсутствует)
    q := query.Get("q")

    // Значение с дефолтом
    category := query.Get("category")
    if category == "" {
        category = "all"
    }

    // Числовые параметры
    page, _ := strconv.Atoi(query.Get("page"))
    if page < 1 {
        page = 1
    }

    perPage, err := strconv.Atoi(query.Get("per_page"))
    if err != nil {
        perPage = 10
    }

    // Проверка наличия параметра
    if query.Has("strict") {  // Go 1.17+
        // strict mode
    }
```

## Скачивание файлов через HTTP

Для скачивания файлов сервер должен установить правильные заголовки и передать содержимое файла.

### Ключевые заголовки
- Content-Type — MIME-тип файла (application/octet-stream для бинарных)
- Content-Disposition — поведение браузера:
  - attachment — скачать файл
  - inline — открыть в браузере
- Content-Length — размер файла (опционально)

### Пример сервера
```go
package main

import (
    "mime"
    "net/http"
    "path/filepath"

    "github.com/gorilla/mux"
)

func downloadHandler(w http.ResponseWriter, r *http.Request) {
    vars := mux.Vars(r)
    filename := vars["filename"]

    // Безопасность: предотвращение directory traversal
    safePath := filepath.Join("./uploads", filepath.Clean(filename))
    absUploads, _ := filepath.Abs("./uploads")
    absPath, _ := filepath.Abs(safePath)

    if !strings.HasPrefix(absPath, absUploads) {
        http.Error(w, "Invalid path", http.StatusBadRequest)
        return
    }

    // Определение MIME-типа
    ext := filepath.Ext(filename)
    contentType := mime.TypeByExtension(ext)
    if contentType == "" {
        contentType = "application/octet-stream"
    }

    // Установка заголовков
    w.Header().Set("Content-Type", contentType)
    w.Header().Set("Content-Disposition", "attachment; filename=\""+filename+"\"")

    // Отправка файла
    http.ServeFile(w, r, safePath)
}

func main() {
    r := mux.NewRouter()
    r.HandleFunc("/download/{filename}", downloadHandler).Methods("GET")

    // Статический файловый сервер
    r.PathPrefix("/static/").Handler(
        http.StripPrefix("/static/", http.FileServer(http.Dir("./static"))),
    )

    http.ListenAndServe(":8080", r)
}
```

### http.ServeFile
http.ServeFile — встроенная функция для отправки файла. Она автоматически:
- Определяет Content-Type по расширению
- Поддерживает Range-запросы (докачка)
- Обрабатывает If-Modified-Since (кэширование)
- Отправляет 404, если файл не найден

### FileServer
```go
// Простой статический сервер
fs := http.FileServer(http.Dir("./static"))
http.Handle("/static/", http.StripPrefix("/static/", fs))
```

### Безопасность
- Всегда проверяйте путь на directory traversal (../)
- Используйте filepath.Clean и проверку префикса
- Не отдавайте файлы за пределами разрешенной директории
- Проверяйте права доступа

## multipart/form-data

multipart/form-data — формат передачи данных, используемый при загрузке файлов через HTML-формы. Данные разделяются на части (parts) с уникальными границами (boundaries).

### Парсинг multipart
```go
func uploadHandler(w http.ResponseWriter, r *http.Request) {
    // Ограничение размера тела запроса
    r.Body = http.MaxBytesReader(w, r.Body, maxUploadSize)

    // ParseMultipartForm парсит тело и сохраняет файлы во временную директорию
    if err := r.ParseMultipartForm(maxUploadSize); err != nil {
        http.Error(w, "File too large", http.StatusBadRequest)
        return
    }
    defer r.MultipartForm.RemoveAll()  // очистка временных файлов

    // Получение текстовых полей
    description := r.FormValue("description")

    // Получение файла
    file, header, err := r.FormFile("file")
    if err != nil {
        http.Error(w, "Error retrieving file", http.StatusBadRequest)
        return
    }
    defer file.Close()

    // Информация о файле
    fmt.Fprintf(w, "File Name: %s\n", header.Filename)
    fmt.Fprintf(w, "File Size: %d\n", header.Size)
    fmt.Fprintf(w, "Content-Type: %s\n", header.Header.Get("Content-Type"))

    // Сохранение файла
    dst, err := os.Create("./uploads/" + header.Filename)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }
    defer dst.Close()

    copied, err := io.Copy(dst, file)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    fmt.Fprintf(w, "Saved %d bytes", copied)
}
```

### Пример сервера
```go
package main

import (
    "fmt"
    "io"
    "net/http"
    "os"

    "github.com/gorilla/mux"
)

const maxUploadSize = 10 * 1024 * 1024  // 10 MB

func uploadHandler(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
        return
    }

    r.Body = http.MaxBytesReader(w, r.Body, maxUploadSize)

    if err := r.ParseMultipartForm(maxUploadSize); err != nil {
        http.Error(w, "File too large or invalid form", http.StatusBadRequest)
        return
    }
    defer r.MultipartForm.RemoveAll()

    description := r.FormValue("description")

    file, header, err := r.FormFile("file")
    if err != nil {
        http.Error(w, "Error retrieving file: "+err.Error(), http.StatusBadRequest)
        return
    }
    defer file.Close()

    // Создание директории, если не существует
    os.MkdirAll("./uploads", 0755)

    dst, err := os.Create("./uploads/" + header.Filename)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }
    defer dst.Close()

    copied, err := io.Copy(dst, file)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    fmt.Fprintf(w, "Upload successful!\nDescription: %s\nFile: %s\nSize: %d bytes",
        description, header.Filename, copied)
}

func main() {
    r := mux.NewRouter()
    r.HandleFunc("/upload", uploadHandler).Methods("POST")
    http.ListenAndServe(":8080", r)
}
```

## JSON-RPC

JSON-RPC — это протокол удаленного вызова процедур (RPC), использующий JSON для кодирования данных. Спецификации: JSON-RPC 1.0 и JSON-RPC 2.0.

### Формат запроса (JSON-RPC 2.0)
```json
{
    "jsonrpc": "2.0",
    "method": "add",
    "params": {"a": 5, "b": 3},
    "id": 1
}
```

### Формат ответа
```json
{
    "jsonrpc": "2.0",
    "result": {"value": 8},
    "id": 1
}
```

### Ручная реализация
```go
package main

import (
    "encoding/json"
    "log"
    "net/http"

    "github.com/gorilla/mux"
)

type JSONRPCRequest struct {
    JSONRPC string      `json:"jsonrpc"`
    Method  string      `json:"method"`
    Params  interface{} `json:"params"`
    ID      interface{} `json:"id"`
}

type JSONRPCResponse struct {
    JSONRPC string      `json:"jsonrpc"`
    Result  interface{} `json:"result,omitempty"`
    Error   interface{} `json:"error,omitempty"`
    ID      interface{} `json:"id"`
}

type AddParams struct {
    A int `json:"a"`
    B int `json:"b"`
}

type AddResult struct {
    Value int `json:"value"`
}

func jsonRPCHandler(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
        return
    }

    var req JSONRPCRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        sendError(w, -32700, "Parse error", nil)
        return
    }

    resp := JSONRPCResponse{
        JSONRPC: "2.0",
        ID:      req.ID,
    }

    switch req.Method {
    case "add":
        paramsMap, ok := req.Params.(map[string]interface{})
        if !ok {
            sendError(w, -32602, "Invalid params", req.ID)
            return
        }
        a := int(paramsMap["a"].(float64))
        b := int(paramsMap["b"].(float64))
        resp.Result = AddResult{Value: a + b}

    case "multiply":
        paramsMap, ok := req.Params.(map[string]interface{})
        if !ok {
            sendError(w, -32602, "Invalid params", req.ID)
            return
        }
        a := int(paramsMap["a"].(float64))
        b := int(paramsMap["b"].(float64))
        resp.Result = AddResult{Value: a * b}

    default:
        sendError(w, -32601, "Method not found", req.ID)
        return
    }

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(resp)
}

func sendError(w http.ResponseWriter, code int, message string, id interface{}) {
    resp := JSONRPCResponse{
        JSONRPC: "2.0",
        Error: map[string]interface{}{
            "code":    code,
            "message": message,
        },
        ID: id,
    }
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusOK)  // JSON-RPC возвращает 200 даже при ошибках
    json.NewEncoder(w).Encode(resp)
}

func main() {
    r := mux.NewRouter()
    r.HandleFunc("/jsonrpc", jsonRPCHandler).Methods("POST")

    log.Println("JSON-RPC Server on :8080")
    log.Fatal(http.ListenAndServe(":8080", r))
}
```

### Коды ошибок JSON-RPC 2.0
- -32700: Parse error
- -32600: Invalid Request
- -32601: Method not found
- -32602: Invalid params
- -32603: Internal error
- -32000 to -32099: Server error


## WebSocket

WebSocket — протокол полнодуплексной связи поверх TCP, обеспечивающий постоянное соединение между клиентом и сервером. В отличие от HTTP (request-response), WebSocket позволяет обеим сторонам отправлять сообщения в любое время.

### Пакет gorilla/websocket
Gorilla WebSocket — самая популярная реализация WebSocket для Go.

### Установка
```bash
go get github.com/gorilla/websocket
```

### Upgrader
Upgrader преобразует HTTP-соединение в WebSocket-соединение.

```go
var upgrader = websocket.Upgrader{
    ReadBufferSize:  1024,
    WriteBufferSize: 1024,
    CheckOrigin: func(r *http.Request) bool {
        return true  // разрешить все источники (только для разработки!)
    },
}
```

### Сервер
```go
package main

import (
    "log"
    "net/http"

    "github.com/gorilla/websocket"
)

var upgrader = websocket.Upgrader{
    ReadBufferSize:  1024,
    WriteBufferSize: 1024,
    CheckOrigin: func(r *http.Request) bool {
        return true
    },
}

func wsHandler(w http.ResponseWriter, r *http.Request) {
    // Upgrade HTTP → WebSocket
    conn, err := upgrader.Upgrade(w, r, nil)
    if err != nil {
        log.Println("Upgrade error:", err)
        return
    }
    defer conn.Close()

    log.Println("Client connected")

    for {
        // Чтение сообщения
        messageType, message, err := conn.ReadMessage()
        if err != nil {
            log.Println("Read error:", err)
            break
        }

        log.Printf("Received: %s", message)

        // Эхо-ответ
        if err := conn.WriteMessage(messageType, message); err != nil {
            log.Println("Write error:", err)
            break
        }
    }

    log.Println("Client disconnected")
}

func main() {
    http.HandleFunc("/ws", wsHandler)
    log.Println("WebSocket server on :8080")
    log.Fatal(http.ListenAndServe(":8080", nil))
}
```

### Типы сообщений WebSocket
- websocket.TextMessage — текстовое сообщение (UTF-8)
- websocket.BinaryMessage — бинарное сообщение
- websocket.CloseMessage — закрытие соединения
- websocket.PingMessage — ping (проверка жизни)
- websocket.PongMessage — pong (ответ на ping)

### GO-клиент
```go

    "time"

            if err != nil {
                log.Println("Write:", err)
                return
            }
            time.Sleep(2 * time.Second)
        }
    }()

    // Чтение ответов
    for {
        _, message, err := c.ReadMessage()
        if err != nil {
            log.Println("Read:", err)
            return
        }
        log.Printf("Received: %s", message)
    }
}
```

## WebDAV

WebDAV (Web Distributed Authoring and Versioning) — расширение протокола HTTP, позволяющее пользователям совместно редактировать и управлять файлами на удаленном веб-сервере. Определен в RFC 4918.

### Методы WebDAV
- PROPFIND — получение свойств ресурса
- PROPPATCH — изменение свойств
- MKCOL — создание коллекции (директории)
- COPY — копирование ресурса
- MOVE — перемещение ресурса
- LOCK — блокировка ресурса
- UNLOCK — снятие блокировки

```go
package main

import (
    "bytes"
    "fmt"
    "io"
    "net/http"
)

func main() {
    baseURL := "http://localhost:8080/webdav/"

    // PUT — создание/замена файла
    req, _ := http.NewRequest("PUT", baseURL+"test.txt", bytes.NewBufferString("Hello WebDAV!"))
    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        panic(err)
    }
    fmt.Println("PUT status:", resp.Status)
    resp.Body.Close()

    // GET — чтение файла
    resp, err = http.Get(baseURL + "test.txt")
    if err != nil {
        panic(err)
    }
    body, _ := io.ReadAll(resp.Body)
    resp.Body.Close()
    fmt.Println("Content:", string(body))

    // PROPFIND — получение свойств
    req, _ = http.NewRequest("PROPFIND", baseURL, nil)
    req.Header.Set("Depth", "1")  // 0=текущий, 1=текущий+дети, infinity=все
    resp, err = http.DefaultClient.Do(req)
    if err != nil {
        panic(err)
    }
    body, _ = io.ReadAll(resp.Body)
    resp.Body.Close()
    fmt.Println("PROPFIND:", string(body))

    // MKCOL — создание директории
    req, _ = http.NewRequest("MKCOL", baseURL+"newfolder/", nil)
    resp, err = http.DefaultClient.Do(req)
    if err != nil {
        panic(err)
    }
    fmt.Println("MKCOL status:", resp.Status)
    resp.Body.Close()

    // DELETE — удаление
    req, _ = http.NewRequest("DELETE", baseURL+"test.txt", nil)
    resp, err = http.DefaultClient.Do(req)
    if err != nil {
        panic(err)
    }
    fmt.Println("DELETE status:", resp.Status)
    resp.Body.Close()

    // COPY — копирование
    req, _ = http.NewRequest("COPY", baseURL+"source.txt", nil)
    req.Header.Set("Destination", baseURL+"dest.txt")
    resp, err = http.DefaultClient.Do(req)
    if err != nil {
        panic(err)
    }
    fmt.Println("COPY status:", resp.Status)
    resp.Body.Close()

    // MOVE — перемещение
    req, _ = http.NewRequest("MOVE", baseURL+"old.txt", nil)
    req.Header.Set("Destination", baseURL+"new.txt")
    resp, err = http.DefaultClient.Do(req)
    if err != nil {
        panic(err)
    }
    fmt.Println("MOVE status:", resp.Status)
    resp.Body.Close()
}
```

## GraphQL

GraphQL — это язык запросов для API и runtime для выполнения этих запросов. Разработан Facebook в 2012, открыт в 2015. В отличие от REST, GraphQL позволяет клиенту запрашивать ровно те данные, которые нужны.

### Основные концепции
- **Schema** — описание типов данных и операций
- **Query** — чтение данных (аналог GET)
- **Mutation** — изменение данных (аналог POST/PUT/DELETE)
- **Subscription** — подписка на события в реальном времени
- **Resolver** — функция, возвращающая данные для поля
- **Type** — объектный, скалярный, перечисляемый, интерфейс, union, input

### Пакет graphql-go

```go
package main

import (
    "encoding/json"
    "log"
    "net/http"

    "github.com/graphql-go/graphql"
    "github.com/gorilla/mux"
)

// Модели
type User struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
    Age  int    `json:"age"`
}

var users = []User{
    {1, "Ivan", 25},
    {2, "Petr", 30},
    {3, "Anna", 22},
}

func main() {
    // Определение типов GraphQL
    userType := graphql.NewObject(graphql.ObjectConfig{
        Name: "User",
        Fields: graphql.Fields{
            "id": &graphql.Field{Type: graphql.Int},
            "name": &graphql.Field{Type: graphql.String},
            "age": &graphql.Field{Type: graphql.Int},
        },
    })

    // Query тип
    rootQuery := graphql.NewObject(graphql.ObjectConfig{
        Name: "Query",
        Fields: graphql.Fields{
            "user": &graphql.Field{
                Type: userType,
                Args: graphql.FieldConfigArgument{
                    "id": &graphql.ArgumentConfig{Type: graphql.Int},
                },
                Resolve: func(p graphql.ResolveParams) (interface{}, error) {
                    id, ok := p.Args["id"].(int)
                    if ok {
                        for _, u := range users {
                            if u.ID == id {
                                return u, nil
                            }
                        }
                    }
                    return nil, nil
                },
            },
            "users": &graphql.Field{
                Type: graphql.NewList(userType),
                Resolve: func(p graphql.ResolveParams) (interface{}, error) {
                    return users, nil
                },
            },
        },
    })

    // Mutation тип
    rootMutation := graphql.NewObject(graphql.ObjectConfig{
        Name: "Mutation",
        Fields: graphql.Fields{
            "createUser": &graphql.Field{
                Type: userType,
                Args: graphql.FieldConfigArgument{
                    "name": &graphql.ArgumentConfig{Type: graphql.NewNonNull(graphql.String)},
                    "age":  &graphql.ArgumentConfig{Type: graphql.NewNonNull(graphql.Int)},
                },
                Resolve: func(p graphql.ResolveParams) (interface{}, error) {
                    user := User{
                        ID:   len(users) + 1,
                        Name: p.Args["name"].(string),
                        Age:  p.Args["age"].(int),
                    }
                    users = append(users, user)
                    return user, nil
                },
            },
        },
    })

    schema, _ := graphql.NewSchema(graphql.SchemaConfig{
        Query:    rootQuery,
        Mutation: rootMutation,
    })

    // HTTP-обработчик
    r := mux.NewRouter()
    r.HandleFunc("/graphql", func(w http.ResponseWriter, r *http.Request) {
        var params struct {
            Query         string                 `json:"query"`
            Variables     map[string]interface{} `json:"variables"`
            OperationName string                 `json:"operationName"`
        }

        if err := json.NewDecoder(r.Body).Decode(&params); err != nil {
            http.Error(w, err.Error(), http.StatusBadRequest)
            return
        }

        result := graphql.Do(graphql.Params{
            Schema:         schema,
            RequestString:  params.Query,
            VariableValues: params.Variables,
            OperationName:  params.OperationName,
        })

        w.Header().Set("Content-Type", "application/json")
        json.NewEncoder(w).Encode(result)
    }).Methods("POST")

    log.Println("GraphQL server on :8080")
    log.Println("Endpoint: POST /graphql")
    log.Fatal(http.ListenAndServe(":8080", r))
}
```

## Пример сервера

```go
package main

import (
    "github.com/kataras/iris/v12"
    swagger "github.com/iris-contrib/swagger/v12"
    "github.com/swaggo/swag"
)

// @title Internet Server Programming API
// @version 1.0
// @description Экзаменационный REST-сервер на Iris с Swagger документацией
// @termsOfService http://swagger.io/terms/
// @contact.name API Support
// @contact.email support@example.com
// @license.name Apache 2.0
// @license.url http://www.apache.org/licenses/LICENSE-2.0.html
// @host localhost:8080
// @BasePath /api/v1
func main() {
    app := iris.New()

    // Группа API
    api := app.Party("/api/v1")
    {
        // @Summary Get all users
        // @Description Get a list of all registered users
        // @Tags users
        // @Produce json
        // @Success 200 {array} User
        // @Router /users [get]
        api.Get("/users", getUsers)

        // @Summary Get user by ID
        // @Description Get a single user by their ID
        // @Tags users
        // @Param id path int true "User ID"
        // @Produce json
        // @Success 200 {object} User
        // @Failure 404 {object} ErrorResponse
        // @Router /users/{id} [get]
        api.Get("/users/{id:int}", getUserByID)

        // @Summary Create a new user
        // @Description Create a new user with the provided data
        // @Tags users
        // @Accept json
        // @Produce json
        // @Param user body User true "User data"
        // @Success 201 {object} User
        // @Failure 400 {object} ErrorResponse
        // @Router /users [post]
        api.Post("/users", createUser)

        // @Summary Update a user
        // @Description Update user data by ID
        // @Tags users
        // @Accept json
        // @Produce json
        // @Param id path int true "User ID"
        // @Param user body User true "User data"
        // @Success 200 {object} User
        // @Failure 400 {object} ErrorResponse
        // @Failure 404 {object} ErrorResponse
        // @Router /users/{id} [put]
        api.Put("/users/{id:int}", updateUser)

        // @Summary Delete a user
        // @Description Delete a user by ID
        // @Tags users
        // @Param id path int true "User ID"
        // @Success 204
        // @Failure 404 {object} ErrorResponse
        // @Router /users/{id} [delete]
        api.Delete("/users/{id:int}", deleteUser)
    }

    // Swagger UI
    config := &swagger.Config{
        URL: "http://localhost:8080/swagger/doc.json",
    }
    swaggerUI := swagger.Handler(swaggerFiles.Handler, config)
    app.Get("/swagger", swaggerUI)
    app.Get("/swagger/{any:path}", swaggerUI)

    app.Run(iris.Addr(":8080"))
}

type User struct {
    ID    int    `json:"id" example:"1"`
    Name  string `json:"name" example:"Ivan Ivanov"`
    Email string `json:"email" example:"ivan@example.com"`
    Age   int    `json:"age" example:"25"`
}

type ErrorResponse struct {
    Error string `json:"error" example:"bad request"`
}

var users = []User{
    {1, "Ivan", "ivan@example.com", 25},
    {2, "Petr", "petr@example.com", 30},
}

func getUsers(ctx iris.Context) {
    ctx.JSON(users)
}

func getUserByID(ctx iris.Context) {
    id := ctx.Params().GetIntDefault("id", 0)
    for _, u := range users {
        if u.ID == id {
            ctx.JSON(u)
            return
        }
    }
    ctx.StatusCode(iris.StatusNotFound)
    ctx.JSON(ErrorResponse{"user not found"})
}

func createUser(ctx iris.Context) {
    var user User
    if err := ctx.ReadJSON(&user); err != nil {
        ctx.StatusCode(iris.StatusBadRequest)
        ctx.JSON(ErrorResponse{err.Error()})
        return
    }
    user.ID = len(users) + 1
    users = append(users, user)
    ctx.StatusCode(iris.StatusCreated)
    ctx.JSON(user)
}

func updateUser(ctx iris.Context) {
    id := ctx.Params().GetIntDefault("id", 0)
    var updated User
    if err := ctx.ReadJSON(&updated); err != nil {
        ctx.StatusCode(iris.StatusBadRequest)
        ctx.JSON(ErrorResponse{err.Error()})
        return
    }
    for i, u := range users {
        if u.ID == id {
            updated.ID = id
            users[i] = updated
            ctx.JSON(updated)
            return
        }
    }
    ctx.StatusCode(iris.StatusNotFound)
    ctx.JSON(ErrorResponse{"user not found"})
}

func deleteUser(ctx iris.Context) {
    id := ctx.Params().GetIntDefault("id", 0)
    for i, u := range users {
        if u.ID == id {
            users = append(users[:i], users[i+1:]...)
            ctx.StatusCode(iris.StatusNoContent)
            return
        }
    }
    ctx.StatusCode(iris.StatusNotFound)
    ctx.JSON(ErrorResponse{"user not found"})
}
```
